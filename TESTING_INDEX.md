## ⚠️ 测试前必读

### 环境要求

**Node.js 版本** ⚠️ **CRITICAL**
```bash
# 检查当前版本
node --version

# 必须是 v20.x.x
# 如果不是 v20，请切换：
nvm install 20
nvm use 20
```

**为什么需要 Node.js 20？**
- 项目使用 ESM 模块系统
- Node.js 24 对 ESM/CJS 互操作更严格
- Electron 是 CommonJS 模块
- Node.js 20 才能正确处理

### 启动应用

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

**预期结果**:
- Electron 窗口自动打开
- 窗口标题显示 "SkillMate"
- 界面正常加载

## 🎯 测试技巧

### 1. 边界测试
- 尝试极端情况（超长文本、超大文件）
- 快速连续操作
- 错误输入（空消息、特殊字符）

### 2. 性能观察
- 打开任务管理器观察内存
- 长时间使用（30 分钟）观察是否变慢
- 检查 CPU 占用

### 3. 用户体验
- 想象自己是新手用户
- 操作是否直观？
- 提示是否清晰？

### 4. 问题记录
- **截图**: 每个问题都截图
- **复现步骤**: 详细记录如何触发
- **预期 vs 实际**: 对比描述

## 📊 测试等级定义

### 优先级分类
- **P0 (致命)**: 应用崩溃，无法使用，必须立即修复
- **P1 (严重)**: 核心功能失效，严重影响使用，应尽快修复
- **P2 (一般)**: 次要功能有问题，不影响主要使用，可延后修复
- **P3 (轻微)**: UI 小瑕疵，不影响功能，低优先级

### 发布标准
- ✅ **可以发布**: P0 = 0, P1 ≤ 1
- ⚠️ **建议修复后发布**: P0 = 0, P1 ≤ 3
- ❌ **不建议发布**: P0 > 0 或 P1 > 3
