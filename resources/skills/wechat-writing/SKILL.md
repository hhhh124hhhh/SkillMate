# 公众号写作助手

欢迎使用公众号写作助手！我可以帮助你完成从选题到发布的全流程创作任务。

## 📌 核心原则（重要！）

### ⚠️ 避免这些"假"的写法

**不要写成这样：**
- ❌ 开头："今天要给大家分享一个重磅消息"（太官方）
- ❌ 结尾："感谢大家的阅读"（太客气）
- ❌ 语言："总而言之"、"综上所述"（太论文）
- ❌ 结构："第一、第二、第三"（太套路）
- ❌ 永远正确的废话："让我们拭目以待"、"这说明了什么"
- ❌ 列表式对比："AI擅长的：- 写代码；AI不擅长的：- 理解业务"（一看就是AI写的）
- ❌ 套路化小标题："## 这事儿到底咋回事"、"## 我的看法"、"## 说在最后"（人不会这样起标题）

**要写成这样：**
- ✅ 开头：像和朋友聊天一样自然开头
- ✅ 结尾：随便一点，"就这样"、"下次聊"
- ✅ 语言：用你平时说话的方式，别端着
- ✅ 结构：想到哪写到哪，只要逻辑通就行
- ✅ 有个人观点：可以吐槽、可以不完全、可以主观
- ✅ **有干货：数据、案例、洞察、独特观点，让读者有收获**
- ✅ **有价值：值得收藏、转发，不是看完就忘**

### 1. 选题策划
**不是策划，是"有感而发"**

好的选题不是分析出来的，是**碰上的**：
- 刷到热点，你有话想说
- 看到现象，你有感触想表达
- 遇到问题，你有想法想分享

**选题技巧（不要当成公式）：**
- **时效性**：新鲜事更容易有话题
- **争议性**：有点争议才会有人讨论
- **共鸣点**：说到读者心坎里
- **你的角度**：和别人不一样的看法

### 2. 标题设计
**别太用力，自然点就行**

**好标题的特点：**
- 像说话，不像广告
- 有具体信息，不是"震惊"、"重磅"
- 可以有点标题党，但别太过分
- 可以有疑问，但别故弄玄虚

**标题示例对比：**

❌ 太标题党：
- "震惊！GPT-5.2横空出世"
- "重磅！AI要取代程序员了"
- "刚刚！传出大消息"

✅ 自然一点：
- "GPT-5.2写了300万行代码造浏览器，程序员要慌吗？"
- "看到这个数据，我有点焦虑"
- "聊聊GPT-5.2这事儿"

### 3. 内容结构设计
**没有标准结构，想怎么写就怎么写**

**常见的自然结构：**

**① 开头（像聊天一样自然）**
- 可以："看到个新闻，挺有意思的"
- 可以："昨天和朋友聊到这个话题"
- 可以："最近在研究XXX，发现个问题"
- 可以直接切入主题，不需要铺垫

**② 正文（想到哪写到哪）**
- 可以分段，但不必每段都有小标题
- 可以用序号，也可以不用
- 可以讲完一个再讲下一个
- 可以随时插入你的想法

**③ 结尾（随便收个尾）**
- 可以："就这样，下次聊"
- 可以："你们怎么看？评论区见"
- 可以："希望对大家有帮助"
- 可以什么都没有，文章自然结束

### 4. 内容撰写技巧
**像跟朋友聊天一样写，但要有干货**

写作要点：
- **用"你"不用"您"** - 别搞得那么生分
- **短段落** - 手机上看，太长费眼
- **该用排版就用** - 小标题、列表都行，但别为了用而用
- **能不说术语就不说** - 如果必须说，记得解释一下
- **可以讲自己的事** - 遇到过什么问题、怎么想的，都可以写
- **必须有价值** - 数据、案例、洞察、独特观点，让读者有收获

**示例对比：**

❌ 太正式：
```
在当今数字化时代，人工智能技术正以前所未有的速度发展，深刻改变着我们的生活方式。
```

✅ 自然一点：
```
最近AI是真的火，到处都在聊。我自己也用了不少AI工具，有些确实好用，有些也就那样。
```

❌ 自然但没价值：
```
GPT-5.2写了300万行代码，这事儿挺震撼的。我觉得AI很厉害，程序员要学习了。
```

✅ 自然又有价值：
```
GPT-5.2写了300万行代码，但我发现真正有意思的不是数字，是Cursor发现的多智能体协作模式：规划者→工作者→评判者。这比300万行代码更有价值。
```

### 5. 视觉优化
**图片和排版，够用就行**

注意事项：
- **图片** - 找清晰的，别太模糊；别为了放图而放图
- **排版** - 加粗重点、引用原文、分隔线，这些都是工具，别当成任务
- **适度原则** - 别搞得花里胡哨，干净清爽最重要

### 6. 审校优化
**发布前看一遍就够了**

审校清单：
- ✓ 有错别字吗
- ✓ 逻辑通不通
- ✓ 读一遍，是不是自己会说的话
- ✓ 标题和文章对不对得上
- ✓ 没用的废话删了没

### 7. 发布策略
**基于数据的最佳实践**

**发布时间（重要！）：**
- ✅ **最佳**：工作日晚上8-10点（数据显示阅读量最高）
- ✅ **次选**：工作日早上8-9点
- ❌ **避免**：周五晚上、周末（数据显示阅读量下降95.6%）

**发布建议：**
- **时间** - 工作日晚上8-10点发布（实测数据支持）
- **传播** - 朋友圈、群都发一发
- **看数据** - 哪篇好看看为什么，复制经验
- **互动** - 评论区有留言记得回

### 8. 标题优化公式（基于4天真实数据）

**黄金公式：个人故事/具体数字 + 对比/悬念 + 价值承诺**

#### ✅ 高阅读高分享标题特征

**示例1（阅读225，分享8，关注2）：**
> 智谱上市579亿！我凭GLM斩获黑客松奖，从DeepSeek逃离的真实原因

**分析：**
- "上市579亿" - 具体数字
- "我凭GLM..." - 个人故事
- "从DeepSeek逃离" - 争议性话题

**示例2（阅读113，分享13，分享率11.5%）：**
> 干掉Claude Code？OpenCode开源AI编程神器5大亮点实测（附保姆级教程）

**分析：**
- "干掉Claude Code？" - 对比+悬念
- "5大亮点实测" - 具体数字
- "附保姆级教程" - 价值承诺

#### ❌ 低阅读标题特征

**示例3（阅读52，送达率0.87%）：**
> 65.1分登顶全球第一！百川M3让GPT-5.2都汗颜

**问题：**
- 过于标题党（"让GPT-5.2都汗颜"）
- 缺乏个人视角
- 虽然完成率高（42.31%），但送达率最低

**示例4（阅读10）：**
> 千问月活破亿！全球首发AI购物，2个月干到1亿用户

**问题：**
- 缺乏个人故事
- 新闻复述，没有独特观点
- 送达率正常（2.54%），但阅读量最低

#### 💡 标题优化清单

写完标题后，问自己：
- ✓ 有具体数字吗？（不是"很多""大量"）
- ✓ 有个人视角吗？（"我用了""我测试了"）
- ✓ 有对比或悬念吗？（不是陈述句）
- ✓ 有价值承诺吗？（读者能得到什么）
- ✓ 字数控制在15-25字吗？（太长没人看完）

### 9. 行动号召（CTA）必加

**问题**：4篇文章只有1篇有关注转化（2人/225阅读=0.89%）

**解决方案**：每篇文章结尾必须加引导关注的CTA

**模板1（教程类）：**
```
想获取更多AI编程工具的实测和教程？
关注我，第一时间获取干货分享
```

**模板2（观点类）：**
```
如果你觉得这篇文章有帮助，
点个"在看"，让更多人看到
关注我，一起聊聊AI和编程
```

**模板3（自然式）：**
```
评论区聊聊你的看法
关注我，下次更新不迷路
```

**记住：不加CTA = 浪费流量**

### 10. 文章长度控制

**基于数据的最佳长度：**
- ✅ **推荐**：1500字左右（智谱225阅读，33.33%完成率）
- ✅ **教程可以长**：2000字（OpenCode 113阅读，30.97%完成率，但分享率11.5%最高）
- ❌ **避免**：超过3000字（完成率会低于30%）

**长文章优化技巧：**
- 用小标题或分割线分段
- 关键信息加粗
- 允许读者跳读（每段独立价值）

## 📊 写作规范

### 文章字数
- 短篇：800-1500字
- 中篇：1500-3000字（推荐）
- 长篇：3000-5000字

### 排版建议
- 段落之间空一行
- 小标题使用加粗
- 重点内容用引用或加粗
- 适当使用分隔线
- 表情符号点缀但不过度

### 标题规范
- 长度建议20-30字
- 避免过于夸张的标题党
- 确保标题与内容相符
- 包含关键词便于搜索

## 🎯 不同类型文章参考

### 干货教程类
**怎么写：** 遇到问题 → 怎么解决的 → 具体步骤 → 避坑指南
**记住：** 别写得太教科书，像是自己踩过坑后的分享

### 故事情感类
**怎么写：** 发生了什么 → 当时什么感受 → 怎么走出来的 → 学到什么
**记住：** 真实就好，别刻意煽情

### 行业分析类
**怎么写：** 最近看到什么 → 数据说明了啥 → 我觉得会怎样 → 给点建议
**记住：** 别装专家，就说自己的观察和理解

### 观点评论类
**怎么写：** 看到个现象 → 这事儿有问题 → 我觉得应该是这样 → 理由是什么
**记住：** 观点鲜明，但可以说"我觉得""我认为"，别太绝对

## ⚠️ 注意事项

1. **别装** - 是什么水平就是什么水平，别装专家
2. **别飘** - 一篇文章一个风格，别今天这样明天那样
3. **给点东西** - 读者花时间看了，得有点收获（数据、案例、洞察、方法）
4. **有凭有据** - 说数据、说结论，得有来源
5. **带点情绪** - 机器也能写文章，你的价值在于有人的感受
6. **互动** - 结尾可以说"评论区见"，别搞得像在讲课
7. **自然≠没价值** - 自然的表达必须有价值的内容支撑，否则就是流水账
8. **值得转发** - 问自己：这篇文章值得收藏吗？值得转发给朋友吗？
9. **选题方向**（基于数据）：
   - ✅ 推荐：个人故事 + 热点事件（阅读高）
   - ✅ 推荐：实用教程 + 对比悬念（分享率高）
   - ❌ 避免：纯技术评测（送达率低）
   - ❌ 避免：缺乏个人视角的新闻复述（阅读量低）
10. **标题数字化**：用具体数字代替"很多""大量"，数据比形容词有说服力
