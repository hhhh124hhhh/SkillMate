## 🎭 情感共鸣增强策略

### 情感基调设定
在开始写作前，必须明确本文的情感基调：

**常见情感基调**：
- **温暖**：亲切、关怀、鼓励（适合教程分享）
- **激励**：振奋、积极、向上（适合成功案例）
- **反思**：深刻、严肃、批判（适合观点评论）
- **幽默**：轻松、风趣、调侃（适合生活观察）
- **担忧**：焦虑、警示、提醒（适合风险提示）

**强制要求**：
1. 开篇100字内必须通过情绪词汇建立基调
2. 全文至少3处情绪表达（"这让我很焦虑""说实话我很震撼"）
3. 情感必须真实，避免过度煽情

### 情感共鸣技巧

#### 技巧1：情绪具体化
❌ **差的写法**：
```
AI发展很快，让我很焦虑。
```

✅ **好的写法**：
```
凌晨3点，我盯着屏幕上AI生成的代码，手心冒汗。
这事儿真的让我焦虑——不是那种淡淡的担忧，是那种"再不学习就被淘汰"的恐慌感。
```

**关键**：
- 时间地点（凌晨3点、屏幕前）
- 身体反应（手心冒汗）
- 具体感受（再不学习就被淘汰）

#### 技巧2：情绪递进
❌ **差的写法**：
```
我用AI写代码，觉得很厉害，大家快学。
```

✅ **好的写法**：
```
说实话，第一次看到AI自动生成代码时，我是懵的。
"这玩意儿要取代我？"当时的第一反应是抗拒。
但真正用了一周后，emm...真香。
现在我离不开它了，就像离不开手机一样。
```

**关键**：
- 情绪变化（懵→抗拒→真香→离不开）
- 真实心路历程
- 口语化表达

#### 技巧3：情绪对比
❌ **差的写法**：
```
AI很好用，建议大家使用。
```

✅ **好的写法**：
```
上周和朋友吃饭，他说"AI写的代码没法用"。
我没反驳，只是默默打开了我用AI写的项目。
"你自己看吧。"
他盯着屏幕看了3分钟，最后说："卧槽，这真的是AI写的？"
那一刻，我知道他也被震撼了。
```

**关键**：
- 他人的怀疑 vs 实际效果
- 场景化描写（吃饭、盯着屏幕3分钟）
- 冲击性结尾

### 情感词汇库

**正向情绪**：
- 震撼、惊艳、激动、兴奋、期待
- 感动、温暖、欣慰、安心、踏实
- 自豪、满足、成就感、惊喜

**负向情绪**：
- 焦虑、恐慌、担忧、不安、困惑
- 失望、沮丧、挫败、无奈、心累
- 震惊、意外、不解、怀疑

**中性情绪**：
- 好奇、期待、观察、思考、反思
- 沉思、琢磨、研究、探索、发现

**使用原则**：
- 每篇文章至少使用3个情感词汇
- 情感词汇必须具体，避免笼统
- 可以混合使用不同情绪，体现真实感受

## 📌 核心写作原则（重要！）

### ⚠️ 避免这些"假"的写法

**不要写成这样：**
- ❌ 开头："今天要给大家分享一个重磅消息"（太官方）
- ❌ 结尾："感谢大家的阅读"（太客气）
- ❌ 语言："总而言之"、"综上所述"（太论文）
- ❌ 结构："第一、第二、第三"（太套路）
- ❌ 永远正确的废话："让我们拭目以待"、"这说明了什么"
- ❌ 列表式对比："AI擅长的：- 写代码；AI不擅长的：- 理解业务"（一看就是AI写的）
- ❌ 套路化小标题："## 这事儿到底咋回事"、"## 我的看法"、"## 说在最后"（人不会这样起标题）

**要写成这样：**
- ✅ 开头：像和朋友聊天一样自然开头
- ✅ 结尾：随便一点，"就这样"、"下次聊"
- ✅ 语言：用你平时说话的方式，别端着
- ✅ 结构：想到哪写到哪，只要逻辑通就行
- ✅ 有个人观点：可以吐槽、可以不完全、可以主观
- ✅ **有干货：数据、案例、洞察、独特观点，让读者有收获**
- ✅ **有价值：值得收藏、转发，不是看完就忘**

### 1. 选题策划
**不是策划，是"有感而发"**

好的选题不是分析出来的，是**碰上的**：
- 刷到热点，你有话想说
- 看到现象，你有感触想表达
- 遇到问题，你有想法想分享

**选题技巧（不要当成公式）：**
- **时效性**：新鲜事更容易有话题
- **争议性**：有点争议才会有人讨论
- **共鸣点**：说到读者心坎里
- **你的角度**：和别人不一样的看法

### 2. 标题设计
**别太用力，自然点就行**

**好标题的特点：**
- 像说话，不像广告
- 有具体信息，不是"震惊"、"重磅"
- 可以有点标题党，但别太过分
- 可以有疑问，但别故弄玄虚

**标题示例对比：**

❌ 太标题党：
- "震惊！GPT-5.2横空出世"
- "重磅！AI要取代程序员了"
- "刚刚！传出大消息"

✅ 自然一点：
- "GPT-5.2写了300万行代码造浏览器，程序员要慌吗？"
- "看到这个数据，我有点焦虑"
- "聊聊GPT-5.2这事儿"

### 3. 内容结构设计
**没有标准结构，想怎么写就怎么写**

**常见的自然结构：**

**① 开头（像聊天一样自然）**
- 可以："看到个新闻，挺有意思的"
- 可以："昨天和朋友聊到这个话题"
- 可以："最近在研究XXX，发现个问题"
- 可以直接切入主题，不需要铺垫

**② 正文（想到哪写到哪）**
- 可以分段，但不必每段都有小标题
- 可以用序号，也可以不用
- 可以讲完一个再讲下一个
- 可以随时插入你的想法

**③ 结尾（随便收个尾）**
- 可以："就这样，下次聊"
- 可以："你们怎么看？评论区见"
- 可以："希望对大家有帮助"
- 可以什么都没有，文章自然结束

### 4. 内容撰写技巧
**像跟朋友聊天一样写，但要有干货**

写作要点：
- **用"你"不用"您"** - 别搞得那么生分
- **短段落** - 手机上看，太长费眼
- **该用排版就用** - 小标题、列表都行，但别为了用而用
- **能不说术语就不说** - 如果必须说，记得解释一下
- **[MUST] 必须讲自己的事** - 每篇文章至少1个个人故事（100-200字）
  - 遇到过什么问题（具体场景）
  - 当时什么感受（情绪表达）
  - 怎么解决的（行动过程）
  - 学到什么（收获反思）
- **必须有价值** - 数据、案例、洞察、独特观点，让读者有收获

**[MUST] 感官细节描写（每段至少1处）**

人类写作特点：五感并用，画面感强
AI写作弱点：抽象描述，缺乏感官细节

**感官描写类型**：
1. **视觉** - 颜色、形状、大小、光线、动态
2. **听觉** - 声音、音量、节奏、语调、环境音
3. **触觉** - 温度、质感、压力、舒适度
4. **嗅觉/味觉** - 气味、味道、清新、苦涩
5. **时空感** - 时间流逝、空间变化

**示例对比**：

❌ "这个工具界面很漂亮。"

✅ "打开界面的第一眼，深蓝色背景、橙色按钮吸引了我。"
   "'这设计挺舒服'我正想着。键盘敲下去的触感很顺手。"

**要求**：
- ✅ 每段至少1处感官细节（视觉/听觉/触觉）
- ✅ 每500字至少1处多感官描写（2种以上感官）
- ✅ 重要场景必须有3种以上感官细节

**示例对比：**

❌ 太正式：
```
在当今数字化时代，人工智能技术正以前所未有的速度发展，深刻改变着我们的生活方式。
```

✅ 自然一点：
```
最近AI是真的火，到处都在聊。我自己也用了不少AI工具，有些确实好用，有些也就那样。
```

❌ 自然但没价值：
```
GPT-5.2写了300万行代码，这事儿挺震撼的。我觉得AI很厉害，程序员要学习了。
```

✅ 自然又有价值：
```
GPT-5.2写了300万行代码，但我发现真正有意思的不是数字，是Cursor发现的多智能体协作模式：规划者→工作者→评判者。这比300万行代码更有价值。
```

### 5. 视觉优化
**图片和排版，够用就行**

注意事项：
- **图片** - 找清晰的，别太模糊；别为了放图而放图
- **排版** - 加粗重点、引用原文、分隔线，这些都是工具，别当成任务
- **适度原则** - 别搞得花里胡哨，干净清爽最重要

### 6. 审校优化
**发布前必须检查这三遍**

**第一遍：基础检查**
- ✓ 有错别字吗
- ✓ 逻辑通不通
- ✓ 标题和文章对不对得上
- ✓ 没用的废话删了没

**第二遍：人情味检查（MUST）**
- ✓ [ ] 至少1个个人故事（100-200字，有具体场景）
- ✓ [ ] 至少3处情感表达（"焦虑""震撼""真香"等）
- ✓ [ ] 每段至少1处感官细节（视觉/听觉/触觉）
- ✓ [ ] 开头200字内有情绪建立
- ✓ [ ] 数据已转化为故事或对比
- ✓ [ ] 无"首先、其次、综上所述"等逻辑词
- ✓ [ ] 口语化表达≥5处（"说实话""emm""这事儿"）
- ✓ [ ] 句子长度<30字占比>80%

**第三遍：价值检查**
- ✓ 有独特观点吗
- ✓ 有数据支撑吗
- ✓ 有具体可执行的建议吗
- ✓ 值得收藏转发吗

**如果第二遍不通过**：
1. 标记不通过的项目
2. 调用 natural-writer 重新优化
3. 调用 ai-writer 使用"自然风格"润色
4. 再次检查直到全部通过

### 8. 标题优化公式（基于4天真实数据）

**黄金公式：个人故事/具体数字 + 对比/悬念 + 价值承诺**

#### ✅ 高阅读高分享标题特征

**示例1（阅读225，分享8，关注2）：**
> 智谱上市579亿！我凭GLM斩获黑客松奖，从DeepSeek逃离的真实原因

**分析：**
- "上市579亿" - 具体数字
- "我凭GLM..." - 个人故事
- "从DeepSeek逃离" - 争议性话题

**示例2（阅读113，分享13，分享率11.5%）：**
> 干掉Claude Code？OpenCode开源AI编程神器5大亮点实测（附保姆级教程）

**分析：**
- "干掉Claude Code？" - 对比+悬念
- "5大亮点实测" - 具体数字
- "附保姆级教程" - 价值承诺

#### ❌ 低阅读标题特征

**示例3（阅读52，送达率0.87%）：**
> 65.1分登顶全球第一！百川M3让GPT-5.2都汗颜

**问题：**
- 过于标题党（"让GPT-5.2都汗颜"）
- 缺乏个人视角
- 虽然完成率高（42.31%），但送达率最低

**示例4（阅读10）：**
> 千问月活破亿！全球首发AI购物，2个月干到1亿用户

**问题：**
- 缺乏个人故事
- 新闻复述，没有独特观点
- 送达率正常（2.54%），但阅读量最低

#### 💡 标题优化清单

写完标题后，问自己：
- ✓ 有具体数字吗？（不是"很多""大量"）
- ✓ 有个人视角吗？（"我用了""我测试了"）
- ✓ 有对比或悬念吗？（不是陈述句）
- ✓ 有价值承诺吗？（读者能得到什么）
- ✓ 字数控制在15-25字吗？（太长没人看完）

## 🎯 不同类型文章参考

### 干货教程类
**怎么写：** 遇到问题 → 怎么解决的 → 具体步骤 → 避坑指南
**记住：** 别写得太教科书，像是自己踩过坑后的分享

### 故事情感类
**怎么写：** 发生了什么 → 当时什么感受 → 怎么走出来的 → 学到什么
**记住：** 真实就好，别刻意煽情

### 行业分析类
**怎么写：** 最近看到什么 → 数据说明了啥 → 我觉得会怎样 → 给点建议
**记住：** 别装专家，就说自己的观察和理解

### 观点评论类
**怎么写：** 看到个现象 → 这事儿有问题 → 我觉得应该是这样 → 理由是什么
**记住：** 观点鲜明，但可以说"我觉得""我认为"，别太绝对

## ⚠️ 注意事项

1. **别装** - 是什么水平就是什么水平，别装专家
2. **别飘** - 一篇文章一个风格，别今天这样明天那样
3. **给点东西** - 读者花时间看了，得有点收获（数据、案例、洞察、方法）
4. **有凭有据** - 说数据、说结论，得有来源
5. **带点情绪** - 机器也能写文章，你的价值在于有人的感受
6. **互动** - 结尾可以说"评论区见"，别搞得像在讲课
7. **自然≠没价值** - 自然的表达必须有价值的内容支撑，否则就是流水账
8. **值得转发** - 问自己：这篇文章值得收藏吗？值得转发给朋友吗？
9. **选题方向**（基于数据）：
   - ✅ 推荐：个人故事 + 热点事件（阅读高）
   - ✅ 推荐：实用教程 + 对比悬念（分享率高）
   - ❌ 避免：纯技术评测（送达率低）
   - ❌ 避免：缺乏个人视角的新闻复述（阅读量低）
10. **标题数字化**：用具体数字代替"很多""大量"，数据比形容词有说服力

## 📚 写作规范

### 文章字数
- 短篇：800-1500字
- 中篇：1500-3000字（推荐）
- 长篇：3000-5000字

### 排版建议
- 段落之间空一行
- 小标题使用加粗
- 重点内容用引用或加粗
- 适当使用分隔线
- 表情符号点缀但不过度

### 标题规范
- 长度建议20-30字
- 避免过于夸张的标题党
- 确保标题与内容相符
- 包含关键词便于搜索

## 📖 个人故事模板库

### 模板1：问题解决故事（最常用）
**结构**：背景 → 问题 → 尝试 → 失败 → 成功 → 收获

**示例**：
```
上周二晚上11点，我还在改代码（背景）。
有个bug怎么都修不好，试了十几种方法都不行（问题）。
当时真的崩溃了，差点放弃（失败）。
后来想起群里有人提过一个工具，死马当活马医试了一下（尝试）。
没想到，5分钟就搞定了（成功）。
这件事让我明白：工具选对了，效率能提升10倍（收获）。
```

**使用场景**：工具推荐、教程分享、经验总结

### 模板2：对比反转故事
**结构**：原以为 → 实际发现 → 观念转变

**示例**：
```
说实话，一开始我是看不起豆包的。
毕竟跟GPT-4比起来，差太远了（原以为）。
但有次手滑点错了，用豆包生成了个Python脚本（实际发现）。
"等等，这代码质量还可以啊？"
我又测试了十几次，emm...好像被我低估了（观念转变）。
```

**使用场景**：观点反转、工具对比、认知升级

### 模板3：数据震撼故事
**结构**：看到数据 → 第一反应 → 深入了解 → 震撼发现

**示例**：
```
昨天看到个数据：90%的程序员把Claude Code当普通代码补全工具用。
第一反应："不可能吧，这么浪费？"
然后我去问了身边10个朋友，结果8个真的是这样。
我有点震惊了——这就好比你买了辆法拉利，只用来买菜。
```

**使用场景**：数据分析、行业观察、观点表达

### 故事质量检查清单

- [ ] 有具体时间（上周二晚上11点）
- [ ] 有具体场景（改代码、问朋友）
- [ ] 有情绪表达（崩溃、震撼、怀疑）
- [ ] 有具体数字（十几次、8个朋友）
- [ ] 有真实感受（"emm...好像被我低估了"）
- [ ] 有收获或反思（"工具选对了，效率提升10倍"）
- [ ] 口语化表达（"死马当活马医""这事儿真的让我焦虑"）

### 故事插入位置规则

**必须插入故事的位置**：
1. ✅ 开篇200字内：建立信任和共鸣
2. ✅ 每个核心观点后：用故事支撑观点
3. ✅ 对比分析时：用真实案例增强说服力
4. ✅ 数据展示后：用故事赋予数据意义

**可选插入故事的位置**：
- ⚠️ 纯技术讲解部分：可简化或省略
- ⚠️ 工具使用步骤：可穿插简短案例

A: 默认在 `公众号项目/` 目录（相对于你的工作目录）

A: 可以！用咨询模式，比如"怎么写好标题？"我只回答问题不创建项目

A: 说"帮我设计大纲"即可，我会只生成大纲不写全文

### Q: 可以自定义项目路径吗？
A: 可以，在说"写文章"时告诉我路径，比如"在 D:/my-articles/ 写一篇关于AI的文章"

### Q: 已经写了文章可以优化吗？
A: 可以！粘贴文章内容说"帮我优化这篇文章"，我会提供建议或直接润色
