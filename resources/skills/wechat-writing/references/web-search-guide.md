## 一、MCP 工具调用

### 1.1 aisearch-mcp-server__chatCompletions

**用途**：搜索实时信息、热点、数据、案例

**调用方式**：
```json
{
  "tool_call": {
    "name": "aisearch-mcp-server__chatCompletions",
    "arguments": {
      "prompt": "[搜索模板]"
    }
  }
}
```

**自动注入**：
- 系统自动注入当前日期到 Prompt 前面
- 格式："【当前日期：2026年01月18日】"
- AI 无需手动调用时间工具

### 1.2 搜索 Prompt 模板库

#### 热点搜索模板
```
请搜索 [主题] 的最新热点，
特别关注最近7天内的事件。

要求：
- 5-10个热点事件
- 每个包含：标题、简述、时间、来源
- 确保时效性
- 结构化返回
```

#### 数据搜索模板
```
请搜索 [主题] 的具体数据，
包括研究报告、统计数据等。

要求：
- 5-10个数据来源
- 每个包含：具体数字、来源、时间
- 优先权威来源
- 结构化返回
```

#### 案例搜索模板
```
请搜索 [主题] 的真实案例，
包括成功案例、实战经验等。

要求：
- 5-10个真实案例
- 每个包含：具体细节、结果、来源
- 优先真实项目经历
- 结构化返回
```

#### 观点搜索模板
```
请搜索 [主题] 的不同观点和争议，
包括专家观点、行业评论、用户反馈等。

要求：
- 3-5个不同观点
- 每个包含：核心论点、支持者、反对意见
- 识别争议焦点
- 结构化返回
```

## 二、搜索策略

### 2.1 关键词设计

**原则**：
- ✅ 具体优于宽泛
- ✅ 加时间范围（"7天内"、"2025年"）
- ✅ 加限定词（"报告"、"案例"、"数据"）
- ❌ 避免过于宽泛（"AI发展"）

**示例**：
| 差的关键词 | 好的关键词 |
|---------|----------|
| "AI工具" | "AI编程工具 2025年 对比" |
| "小程序" | "小程序开发 最新数据 2025" |
| "写作" | "AI写作工具 评测 2025" |
| "GPT" | "GPT-5 具体参数 基准测试" |

### 2.2 搜索顺序

**推荐顺序**：
1. 热点事件（时效性优先）
2. 数据和案例（支撑内容）
3. 观点和争议（独特性）

### 2.3 结果验证

**真实性检查**：
- 来源可靠吗？（官方、研究机构）
- 数据具体吗？（有数字，不是"很多"）
- 时间明确吗？（有日期，不是"最近"）

**独特性检查**：
- 有不同观点吗？
- 有争议点吗？
- 能找到差异化角度吗？

## 三、降级处理

### 3.1 MCP 失败处理

```
1. 自动重试 2 次（间隔 2 秒）
2. 降级到 AI 通用知识 + 明确标注
3. 提供可操作建议
```

### 3.2 友好错误提示

```
❌ 搜索服务暂时不可用

⚠️ 基于我的知识，我推荐以下内容：
（标注：基于 AI 通用知识，可能不是最新）

建议：
1. 检查网络连接
2. 稍后重试（5-10分钟后）
3. 手动提供资料，我帮你整理
```

## 四、最佳实践

### DO（推荐做法）
✅ 使用结构化 Prompt 模板
✅ 要求具体时间范围
✅ 明确数据来源
✅ 多角度搜索（热点、数据、观点）
✅ 交叉验证关键数据

### DON'T（避免做法）
❌ 使用过于宽泛的关键词
❌ 不要求时间范围
❌ 忽略数据来源
❌ 只搜索一次
❌ 盲目相信搜索结果

## 五、质量检查清单

### 搜索完成后检查

**数据来源**：
- [ ] 至少 3 个真实数据来源
- [ ] 所有数据注明来源
- [ ] 来源可靠性验证（官方、研究机构）

**真实案例**：
- [ ] 至少 2 个真实案例
- [ ] 案例有具体细节
- [ ] 案例有明确结果

**独特观点**：
- [ ] 至少 1 个独特观点
- [ ] 观点与常见观点不同
- [ ] 观点有逻辑支撑

**时效性**：
- [ ] 所有信息有时间戳
- [ ] 热点事件在 7 天内
- [ ] 数据在 1 年内

## 六、常见问题

### Q1: MCP 搜索超时怎么办？
A: 系统会自动重试 2 次，仍失败则降级到 AI 通用知识

### Q2: 搜索结果不准确怎么办？
A: 优化搜索关键词，增加 "2025年"、"最新" 等限定词

### Q3: 如何确保数据真实性？
A: 检查来源是否权威（官方、研究机构），要求具体数字

### Q4: 如何找到独特观点？
A: 搜索 "不同观点"、"争议"、"专家观点"，对比分析

### Q5: 搜索结果太少怎么办？
A: 放宽关键词范围，减少限定词，或使用同义词

### Q6: 如何避免重复内容？
A: 多次搜索不同角度，交叉验证，合并相似结果

## 七、实战示例

### 示例1：AI 工具评测文章

**搜索计划**：
1. 热点搜索："AI编程工具 最新 2025年 对比"
2. 数据搜索："AI编程工具 性能基准测试 具体数据"
3. 案例搜索："AI编程工具 真实案例 项目经验"
4. 观点搜索："AI编程工具 争议 专家观点"

**预期结果**：
- 5-10 个热点事件（新版本发布、重大更新）
- 3+ 个数据来源（基准测试成绩、性能指标）
- 2+ 个真实案例（真实项目使用经验）
- 3-5 个不同观点（支持者、反对者、中立观点）

### 示例2：行业趋势分析文章

**搜索计划**：
1. 热点搜索："[行业] 最新趋势 2025年 报告"
2. 数据搜索："[行业] 市场数据 增长率 统计"
3. 案例搜索："[行业] 成功案例 企业实践"
4. 观点搜索："[行业] 未来预测 专家观点"

**预期结果**：
- 5-10 个热点事件（行业新闻、政策变化）
- 3+ 个数据来源（市场规模、增长率、预测数据）
- 2+ 个真实案例（领先企业实践）
- 3-5 个不同观点（乐观、悲观、中立预测）
