# 内容创作工作流详解

本文档详细说明从零开始创作公众号文章的完整工作流程，包括每个阶段的步骤、Skills调用和输出要求。

---

## 🎯 工作流概述

### 目标
从零开始创作一篇高质量的公众号文章，确保：
- 选题有热点和争议性
- 标题有吸引力和点击率
- 结构清晰逻辑连贯
- 内容有价值和深度
- 排版美观易读

### 适用场景
- 全新文章创作
- 深度技术文章
- 热点事件评论
- 项目复盘总结

### 预计时间
- 资料收集：1-2小时
- 大纲设计：30-60分钟
- 内容创作：2-4小时
- 排版优化：30-60分钟
- **总计**：4-8小时

## 📊 2026年开篇设计详解

### 开篇的重要性（2026年标准）

**前3行决定生死**：
- 前3行必须包含关键词和利益点
- 100字内必须制造"开幕雷击"效果
- 200字内必须建立信任和连接

### 组合式开篇结构（推荐）

```
┌─────────────────────────────────────────┐
│ 热点切入（前3行，50-80字）               │
│ - 包含关键词：智谱上市、579亿市值        │
│ - 制造紧迫感：当天、上市、市值          │
│ - 建立相关性：对个人意味着什么          │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│ 金句观点（100字内）                     │
│ - 制造震撼："认知上限决定AI使用上限"   │
│ - 建立深度：AI是乘法不是加法           │
│ - 引发好奇：为什么说乘法？             │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│ 个人故事（连接部分，50-100字）          │
│ - 建立信任：黑客松获奖                 │
│ - 情感共鸣：真实经历                   │
│ - 自然过渡：引出文章主题               │
└─────────────────────────────────────────┘
```

### 开篇实例分析

**智谱上市文章开篇（200字）**：

```markdown
2026年1月8日，智谱AI正式登陆港交所，上市首日市值达到579亿港元。
（热点切入：关键词、紧迫感、相关性）

这个消息让我想起了3个月前在黑客松现场听到的一句话：
（过渡连接）

**"认知上限决定AI使用上限。AI对个人是乘法，不是加法。"**
（金句观点：震撼、深度、好奇）

当时我正站在AI切磋大会第18期·WaytoAGI全球黑客松的领奖台上，手里捧着二等奖的证书——我的Nexus财务智能体项目，正是凭借GLM的帮助才得以完成。
（个人故事：信任、共鸣、过渡）

今天我想分享这个完整的故事：从DeepSeek到GLM的迁移历程，以及在AI时代，为什么选择比努力更重要。
```

**为什么这个开篇有效？**
- ✅ 前3行包含关键词（智谱上市、579亿）
- ✅ 100字内有"开幕雷击"（金句）
- ✅ 200字内建立信任（获奖、GLM帮助）
- ✅ 自然过渡到主题（分享故事）

### 开篇常见错误（2026年已淘汰）

**❌ 错误1：热点切入太长**
```markdown
2026年1月8日上午9点30分，北京智谱华章科技股份有限公司（股票代码：2513.HK）正式在香港联合交易所挂牌上市...（200字还在讲上市）
```
**问题**：前3行没有核心关键词，读者等不到重点

**❌ 错误2：直接开始讲故事**
```markdown
三个月前我参加了一个黑客松比赛，获得了二等奖...
```
**问题**：缺少热点和金句，无法吸引点击

**❌ 错误3：使用"一、二、三"标题**
```markdown
## 一、上市新闻
## 二、金句观点
## 三、个人经历
```
**问题**：2026年已淘汰这种格式，影响阅读体验，应使用自然分隔

## 🔧 阶段详解

### 阶段0：开篇设计（新增，2026年重点）

**目标**：设计符合2026年爆款标准的开篇（200字）

**步骤**：
1. **热点切入**（50-80字）
   - 前3行必须包含关键词
   - 制造紧迫感和时效性
   - 建立与读者的相关性

2. **金句观点**（50-100字）
   - 制造"开幕雷击"效果
   - 建立深度思考预期
   - 引发读者好奇心

3. **个人故事**（50-100字）
   - 建立信任和情感共鸣
   - 自然过渡到文章主题
   - 为全文奠定基调

**输出**：
- ✅ 200字精彩开篇
- ✅ 点击率、完读率、传播率三重保证

---

### 阶段1：项目初始化

**目标**：创建标准项目结构

**步骤**：
1. 用户输入文章标题或主题
2. 系统创建项目文件夹
3. 生成标准文件结构：
   - `00_背景资料.md`
   - `01_文章大纲.md`
   - `02_文章内容.md`
   - `03_发布记录.md`
   - `assets/`目录

**输出**：
- ✅ 项目文件夹创建完成
- ✅ 文件结构初始化完成

---

### 阶段2：资料收集（联网搜索增强版）

**目标**：收集充分的背景资料，强调实时性和真实性

**调用Skill**：`topic-selector`（热点搜索） + `aisearch-mcp-server__chatCompletions`（深度资料收集）

**[MUST] 联网搜索步骤**（新增）：

1. **热点事件搜索**（使用 MCP）
   - 调用：aisearch-mcp-server__chatCompletions
   - 搜索："[主题] 最新热点 7天内"
   - 质量要求：5-10个热点，每个含来源链接

2. **技术数据搜索**（使用 MCP）
   - 搜索1："[主题] 具体数据 研究报告"
   - 搜索2："[主题] 对比分析 竞品数据"
   - 质量要求：3+个数据来源，必须有具体数字

3. **用户案例搜索**（使用 MCP）
   - 搜索："[主题] 真实案例 成功经验"
   - 质量要求：2+个真实案例，必须有具体细节

4. **观点和争议搜索**（使用 MCP）
   - 搜索："[主题] 不同观点 争议"
   - 质量要求：3-5个不同观点

5. **真实性检查**
   - 验证所有数据来源
   - 标注具体来源链接
   - 统计达标情况

6. **独特性验证**
   - 对比不同观点
   - 识别差异化角度
   - 提炼独特观点

**步骤**：
1. **热点事件追踪**
   - 搜索最新热点（7天内）
   - 识别高关注度话题
   - 收集关键数据和事实

2. **技术数据收集**
   - 技术指标对比
   - 基准测试成绩
   - 价格和性价比分析

3. **参考资料整理**
   - 媒体报道链接
   - 官方文档和数据
   - 社区讨论热点

4. **用户案例收集**
   - 真实项目经历
   - 获奖或成就
   - 具体数据和结果

**输出**：
- 📄 `00_背景资料.md` 充实完整

**质量标准**（更新）：
- ⭐ 至少 3 个真实数据来源（必须有来源链接）
- ⭐ 至少 2 个真实案例（必须有具体细节）
- ⭐ 至少 1 个独特观点（必须与常见观点不同）
- ⭐ 所有数据注明来源
- ⭐ 所有案例有时间戳

---

### 阶段3：标题设计

**目标**：生成高点击率标题

**调用Skill**：`title-generator`

**步骤**：
1. **分析背景资料**
   - 提取核心关键词
   - 识别热点元素
   - 找出用户独特价值

2. **生成候选标题**
   - 10+个不同角度的标题
   - 多种风格（数据冲击、情绪张力、争议性等）

3. **标题评分**
   - 预测点击率
   - 识别最佳3个

4. **A/B测试建议**
   - 标题变体
   - 发布时机建议

**输出**：
- 📄 在 `01_文章大纲.md` 中记录候选标题
- 📄 在 `02_文章内容.md` 中选择主标题

**质量标准**：
- 至少10个候选标题
- 标题包含3+关键词
- 有数据或情绪冲击力

---

### 阶段4：结构设计

**目标**：设计清晰的文章框架

**调用Skill**：`wechat-writing`

**步骤**：
1. **确定文章类型**
   - 干货教程类
   - 故事情感类
   - 行业分析类
   - 观点评论类

2. **设计7部分框架**
   - 一、开头（400字）
   - 二、项目介绍（600字）
   - 三、问题分析（700字）
   - 四、深度体验（800字）
   - 五、对比分析（600字）
   - 六、实用建议（500字）
   - 七、总结（300字）

3. **分配字数**
   - 总字数3900字
   - 各部分字数和占比

4. **提炼核心观点**
   - 每部分的核心论点
   - 支撑数据和案例
   - 逻辑衔接

**输出**：
- 📄 `01_文章大纲.md` 详细完整

**质量标准**：
- 7部分结构清晰
- 每部分有核心观点
- 逻辑连贯无断层
- 字数分配合理

---

### 阶段5：风格匹配

**目标**：确保文章符合用户风格

**调用Skill**：`style-learner`

**步骤**：
1. **分析用户历史文章**
   - 读取5-10篇历史文章
   - 提取风格特征

2. **风格维度分析**
   - 标题特征（痛点+解决方案、数据冲击）
   - 结构特征（热点引入→干货→建议）
   - 语言特征（真实、硬核、实用）
   - 情绪特征（"炸裂"、"超神"等）

3. **生成风格指南**
   - 用词偏好
   - 句式特点
   - 段落长度
   - 标点使用

4. **应用风格指南**
   - 在创作时遵循风格
   - 确保一致性

**输出**：
- 📄 风格分析报告（可选）
- 📄 在 `01_文章大纲.md` 中记录风格要点

**质量标准**：
- 风格识别准确
- 与历史文章一致
- 保持个人特色

---

### 阶段6：内容创作

**目标**：撰写完整文章内容

**调用Skill**：`ai-writer`

**步骤**：
1. **撰写开头（400字）**
   - 热点引入（数据/事件）
   - 个人经历（获奖/成就）
   - 价值预告（读者将获得什么）

2. **撰写正文（2500字）**
   - 项目介绍（600字）
     - 项目背景
     - 核心创新
     - 项目成果数据
   - 问题分析（700字）
     - 遇到的具体问题
     - 转折点
     - 决策过程
   - 深度体验（800字）
     - 技术突破
     - 实际使用体验
     - 具体案例
   - 对比分析（600字）
     - 对比表格
     - 适用场景
     - 选择建议

3. **撰写建议（500字）**
   - 选型建议
   - 适用场景
   - 未来展望

4. **撰写结尾（300字）**
   - 个人总结
   - 核心观点
   - 互动引导

5. **多轮润色**
   - 检查逻辑连贯性
   - 优化语言表达
   - 补充数据支撑
   - 删除冗余内容

**输出**：
- 📄 `02_文章内容.md` 完整文章

**质量标准**：
- 总字数3900字左右
- 逻辑清晰无断层
- 数据准确有来源
- 语言流畅易读

---

### 阶段7：排版优化

**目标**：提升文章视觉效果

**调用Skill**：`smart-layout`

**步骤**：
1. **选择排版模板**
   - 简约商务
   - 清新文艺
   - 科技感
   - 专业学术

2. **优化视觉层次**
   - 小标题分隔
   - 段落间距
   - 重点加粗
   - 引用强调

3. **插入图片和图表**
   - 封面图（900x500px）
   - 数据图表（对比表格、趋势图）
   - 流程图/架构图
   - 截图/照片

4. **格式检查**
   - 段落不超过5-6行
   - 适当使用列表
   - 合理使用分隔线
   - 表情符号点缀

**输出**：
- 📄 `02_文章内容.md` 排版优化完成
- 📁 `assets/` 素材齐全

**质量标准**：
- 视觉层次清晰
- 易于手机阅读
- 图片质量高
- 格式规范统一

---

### 阶段8：数据录入

**目标**：归档项目数据

**调用Skill**：`data-writer`

**步骤**：
1. **录入项目数据**
   - 文章标题和类型
   - 创作时间
   - 字数统计
   - 使用的skills

2. **更新文章统计**
   - 背景资料数量
   - 参考链接数量
   - 图片素材数量
   - 核心数据指标

3. **归档参考资料**
   - 保存所有引用链接
   - 记录数据来源
   - 备份重要文档

**输出**：
- 📄 `03_发布记录.md` 初始化完成
- 📄 项目数据录入完成

**质量标准**：
- 数据完整准确
- 来源清晰可追溯
- 格式规范统一

## 🎨 不同类型文章的调整

### 干货教程类

**调整要点**：
- 增加"实操步骤"部分
- 补充"常见问题"部分
- 提供"资源下载"链接

**结构调整**：
```
开头 → 痛点分析 → 解决方案 → 实操步骤 → 案例分析 → 常见问题 → 总结
```

### 故事情感类

**调整要点**：
- 强调情感冲突
- 增加细节描写
- 提升共鸣点

**结构调整**：
```
开头 → 冲突 → 转折 → 高潮 → 反思 → 启示 → 互动
```

### 行业分析类

**调整要点**：
- 增加数据图表
- 提供趋势预测
- 给出专业建议

**结构调整**：
```
开头 → 背景分析 → 数据解读 → 趋势预测 → 战略建议 → 风险提示 → 总结
```
