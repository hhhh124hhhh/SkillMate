# 公众号封面设计指南

本文档整合微信公众号封面的所有特殊要求，为 AI 生成提示词提供完整的指导。

---

## 一、核心要求概述

### 1.1 比例要求

**标准比例**：2.35:1（横版）

**推荐尺寸**：
- 标准尺寸：900 × 383 像素
- 大尺寸：1080 × 460 像素
- 高清尺寸：1920 × 817 像素

**重要提示**：
- ❌ 不要使用正方形（1:1）
- ❌ 不要使用竖版（如 3:4）
- ✅ 必须使用 2.35:1 横版

### 1.2 双重场景设计

**关键点**：同一张图片会用于两个场景
1. **头条封面展示**：2.35:1 横版完整显示
2. **分享卡片**：自动裁剪为 1:1 正方形

**设计含义**：
- 必须确保裁剪为 1:1 后，核心内容依然完整
- 所有关键元素必须集中在中心区域
- 四周区域只能放装饰性元素

**裁剪示意图**：
```
原始封面 (2.35:1)              分享卡片 (1:1)
┌────────────────────┐         ┌──────────┐
│  ❌边缘内容被切掉  │         │          │
│ ┌──────────────┐  │   →     │  ✅核心  │
│ │  ✅核心内容  │  │         │  内容    │
│ └──────────────┘  │         │          │
│  ❌边缘内容被切掉  │         └──────────┘
└────────────────────┘
```

## 二、安全区域设计

### 2.1 中心 60% 原则

**核心原则**：所有关键内容必须放在画面中心 60% 的区域

**计算方法**：
- 画面宽度：900px
- 安全区域：900 × 60% = 540px
- 左右边距各：20%（180px）

**安全区域图示**：
```
┌─────────────────────────────────────┐ 900px
│ ⚠️  左侧危险区 (180px)  │  ⚠️右侧危险区 (180px)
│ ┌───────────────────┐               │
│ │  ✅ 安全区 (540px) │              │
│ │                   │              │
│ │   主标题          │              │
│ │   副标题          │              │
│ │   核心图标        │              │
│ │                   │              │
│ └───────────────────┘               │
└─────────────────────────────────────┘
```

**安全区域规则**：
1. ✅ 主标题必须在安全区中心
2. ✅ 副标题必须在安全区内
3. ✅ 核心图标必须在安全区内
4. ⚠️ 装饰元素可以放在危险区
5. ❌ 重要文字不能放在危险区

### 2.2 文字和元素的安全距离

**最小距离**：
- 文字距离边缘 ≥ 15%（135px）
- 关键元素距离边缘 ≥ 20%（180px）

**推荐距离**：
- 主标题距离边缘 ≥ 20%（180px）
- 副标题距离边缘 ≥ 18%（162px）
- 核心图标距离边缘 ≥ 20%（180px）

**测试方法**：
```python
# 验证文字是否在安全区域
def is_in_safe_area(x, width=900):
    """检查元素是否在安全区域"""
    safe_area = width * 0.6
    margin = (width - safe_area) / 2
    return margin <= x <= (width - margin)
```

### 2.3 防裁剪设计清单

**必须检查**：
- [ ] 主标题中心是否在画面正中央？
- [ ] 副标题是否在主标题下方且居中？
- [ ] 核心图标是否在中心区域？
- [ ] 是否有文字延伸到左右边缘？
- [ ] 是否有关键图标在边缘位置？
- [ ] 裁剪为正方形后标题是否完整？
- [ ] 裁剪为正方形后图标是否可见？

## 三、内容布局规范

### 3.1 标准布局结构

```
┌───────────────────────────────────────┐
│         [顶部留白 5-10%]               │
│                                       │
│              主标题                    │  ← 最大字号
│           （核心信息）                 │     居中
│                                       │
│              副标题                    │  ← 中等字号
│         （补充说明）                   │     居中
│                                       │
│           [核心图标]                   │  ← 视觉焦点
│         （可选元素）                   │     居中
│                                       │
│         [底部留白 5-10%]               │
└───────────────────────────────────────┘
```

### 3.2 标题设计要求

#### 主标题
- **位置**：画面正中央
- **字号**：最大（建议占画面宽度的 15-20%）
- **颜色**：与背景对比度 ≥ 7:1
- **字数**：建议 ≤ 10 个字
- **示例**：
  - ✅ "智谱上市579亿"
  - ❌ "智谱AI正式上市，估值达到579亿，成为AI领域新标杆"（过长）

#### 副标题
- **位置**：主标题正下方
- **字号**：主标题的 50-60%
- **颜色**：与主标题同色或略浅
- **字数**：建议 ≤ 15 个字
- **示例**：
  - ✅ "GLM-4.7实测"
  - ✅ "从入门到精通"
  - ❌ "全面解析GLM-4.7的强大功能和实际应用场景"（过长）

### 3.3 核心图标设计

**位置**：主标题和副标题之间或周围

**尺寸**：占画面高度的 20-30%

**风格要求**：
- 扁平化设计
- 简洁易识别
- 与主题相关
- 与文字形成平衡构图

**常见图标类型**：
- 技术类：芯片、代码符号 `</>`、AI 大脑
- 成长类：上升箭头 `↑`、图表、奖杯
- 生活类：书本、灯泡、心形

**示例布局**：
```
┌─────────────────────────────────────┐
│                                     │
│      [AI芯片图标]                   │  ← 图标在上
│      智谱上市579亿                  │
│      GLM-4.7实测                    │
│                                     │
└─────────────────────────────────────┘

或

┌─────────────────────────────────────┐
│                                     │
│      智谱上市579亿                  │
│  </>  GLM-4.7实测  </>            │  ← 图标在两侧
│                                     │
└─────────────────────────────────────┘
```

## 四、背景和装饰设计

### 4.1 背景设计

**推荐方案**：
1. **渐变背景**（首选）
   - 从深色到浅色的柔和渐变
   - 避免过于鲜艳的颜色
   - 确保与文字对比度足够

2. **纯色背景**
   - 使用低饱和度的颜色
   - 深色背景配白色文字
   - 浅色背景配深色文字

3. **纹理背景**
   - 轻微的几何纹理
   - 不干扰主要内容
   - 保持低对比度

**避免的设计**：
- ❌ 复杂的图案（影响文字可读性）
- ❌ 高对比度的渐变（视觉疲劳）
- ❌ 过于花哨的背景（分散注意力）

### 4.2 装饰元素

**可以放在危险区的装饰**：
- 小圆点、光点
- 几何线条
- 淡化的图标水印
- 渐变光晕

**装饰原则**：
- 装饰元素不能分散注意力
- 保持低透明度（30-50%）
- 不影响文字可读性
- 裁剪后不影响整体效果

**示例**：
```
┌─────────────────────────────────────┐
│ • • • │  标题内容  │ • • •          │  ← 点状装饰
│       │  （中心）   │                │
│━━━━━━━│─────────────│━━━━━━━━━      │  ← 线条装饰
└─────────────────────────────────────┘
```

```

```

文字：#065F46（深绿）
对比度：8.7:1 ✅ 优秀
```

### 5.2 对比度要求

**WCAG 标准**：
- AAA级：对比度 ≥ 7:1（强烈推荐）
- AA级：对比度 ≥ 4.5:1（最低要求）

**测试方法**：
1. 使用在线对比度检查器
2. 在不同设备上测试
3. 考虑不同光线条件
4. 考虑色盲用户

**常见错误**：
- ❌ 浅灰色文字 + 浅色背景
- ❌ 红色文字 + 绿色背景
- ❌ 相近颜色的搭配

```

```

```

```

```

```

```

```
生成 1024×1024 的图片
（不符合微信要求的 2.35:1）
```

```
生成 900×383 的图片
```

```
浅灰色文字 (#CCCCCC) + 浅灰色背景 (#EEEEEE)

```
**✅ 正确做法**：提高对比度
```
白色文字 (#FFFFFF) + 深蓝色背景 (#1E3A8A)
对比度：11.2:1 ✅ 优秀
```

def test_square_crop(image_path):
    """测试裁剪为正方形后的效果"""
    img = Image.open(image_path)
    width, height = img.size

    # 计算裁剪区域（中心正方形）

    right = left + height
    cropped = img.crop((left, 0, right, height))

### 9.2 对比度测试

**在线工具**：
- WebAIM Contrast Checker: https://webaim.org/resources/contrastchecker/
- Contrast Ratio: https://contrast-ratio.com/

**Python 测试代码**：
```python
def calculate_contrast_ratio(hex_color1, hex_color2):
    """计算两个颜色的对比度"""
    def hex_to_rgb(hex_color):
        hex_color = hex_color.lstrip('#')
        return tuple(int(hex_color[i:i+2], 16) for i in (0, 2, 4))

    def get_luminance(rgb):
        r, g, b = rgb
        rs, gs, bs = map(lambda x: x / 12.92 if x <= 0.03928 else ((x + 0.055) / 1.055) ** 2.4, (r, g, b))
        return 0.2126 * rs + 0.7152 * gs + 0.0722 * bs

    rgb1 = hex_to_rgb(hex_color1)
    rgb2 = hex_to_rgb(hex_color2)

    l1 = get_luminance(rgb1)
    l2 = get_luminance(rgb2)

    lighter = max(l1, l2)
    darker = min(l1, l2)

    contrast_ratio = (lighter + 0.05) / (darker + 0.05)
    return contrast_ratio

# 使用示例
ratio = calculate_contrast_ratio('#FFFFFF', '#1E3A8A')
print(f"对比度: {ratio:.2f}:1")
if ratio >= 7:
    print("✅ AAA级（优秀）")
elif ratio >= 4.5:
    print("⚠️ AA级（合格）")
else:
    print("❌ 不合格")
```

## 十一、最佳实践

### 11.1 设计原则

1. **简洁至上**：少即是多，避免过度设计
2. **内容为王**：设计服务于内容，不要喧宾夺主
3. **用户视角**：从读者角度审视设计
4. **测试驱动**：多设备、多场景测试
5. **数据驱动**：根据数据反馈优化设计

### 11.2 性能优化

1. **文件大小**：
   - 尽量控制在 500KB 以内
   - 使用适当的压缩
   - 避免过度精细的细节

2. **加载速度**：
   - 使用 JPG 格式（照片类）

### 11.3 可访问性

1. **对比度**：确保色盲用户可读
2. **文字大小**：不过小，不过大
3. **色彩选择**：不仅依赖颜色传达信息
4. **测试工具**：使用可访问性测试工具

## 十二、常见问题 FAQ

### Q1: 为什么必须使用 2.35:1 比例？

**A**: 这是微信官方规定的封面比例，使用其他比例会导致：
- 显示不完整
- 被自动裁剪
- 显示异常

### Q2: 如何确保裁剪后内容完整？

**A**:
1. 所有关键内容放在中心 60% 区域
2. 手动裁剪测试
3. 使用检查清单验证
4. 多设备测试

### Q3: 可以在边缘放文字吗？

**A**:
- ❌ 重要文字不能放在边缘
- ✅ 装饰性小文字可以（如标签、日期）
- ⚠️ 但要确保裁剪后不影响核心信息

### Q4: 如何选择配色方案？

**A**:
1. 根据内容主题选择（科技/生活/情感）
2. 确保对比度 ≥ 7:1
3. 保持品牌一致性
4. 测试色盲用户的可读性

### Q5: 封面和配图风格如何统一？

**A**:
1. 使用相同的配色方案
2. 使用相同的装饰元素风格
3. 保持相似的视觉语言
4. 调整比例适配不同场景
