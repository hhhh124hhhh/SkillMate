# 豆包Seedream生图最佳实践指南

本文档总结使用火山引擎豆包Seedream 4.5进行AI图像生成的最佳实践。

---

## 📐 Prompt工程原则

### 基础公式

**公式**：主体 + 场景 + 风格

**示例**：
```
主体：技术架构图
场景：展示微服务系统结构
风格：扁平化科技风格

组合后：
技术架构图，展示微服务系统结构
扁平化科技风格，蓝色和橙色配色
```

---

### 进阶公式

**公式**：主体描述 + 场景描述 + 风格定义 + 镜头语言 + 氛围词 + 细节修饰

**示例**：
```
主体描述：技术架构图，展示微服务的3层架构
场景描述：从上到下依次是API网关、业务服务、数据层
风格定义：扁平化设计，科技感，现代简约风格
镜头语言：横版布局，16:9比例
氛围词：清晰易读，专业
细节修饰：箭头表示数据流向，使用蓝色和橙色配色方案，白色背景
```

---

### 三大核心原则

#### 1. 具体性原则

**❌ 错误示例**：
```
一张好看的架构图
```

**✅ 正确示例**：
```
技术架构图，展示微服务系统的3层架构
从上到下依次是API网关、业务服务、数据层
使用扁平化设计，蓝色和橙色配色
箭头表示HTTP请求和数据流向
白色背景，16:9比例
```

**说明**：描述越具体，生成结果越精确。

---

#### 2. 结构化原则

**推荐结构**：
1. 主体（是什么）
2. 场景（展示什么内容）
3. 布局（如何组织）
4. 风格（视觉风格）
5. 细节（颜色、比例、背景）

**示例**：
```
# 1. 主体
技术对比表格

# 2. 场景
对比GLM-4.7和DeepSeek两个AI工具

# 3. 布局
使用表格形式，包含5个对比维度：
代码生成质量、稳定性、Agent能力、价格、适用场景

# 4. 风格
专业商业风格，现代化配色方案

# 5. 细节
使用交替行颜色，勾叉评分，16:9比例
```

---

#### 3. 细节丰富性原则

**必须包含的细节**：
- **颜色**：蓝色、绿色、橙色、渐变色
- **布局**：横版、竖版、卡片式、分栏
- **比例**：16:9、1:1、9:16
- **背景**：白色、深色、渐变
- **风格**：扁平化、科技感、现代简约、专业

**示例**：
```
技术指标卡片图
- 使用3个卡片横向排列
- 卡片1：蓝色渐变
- 卡片2：绿色渐变
- 卡片3：橙色渐变
- 白色背景
- 16:9比例
- 扁平化设计
- 现代科技风格
```

## 🎯 场景适用性指南

### 营销推广类场景（AI强烈推荐 ⭐⭐⭐⭐⭐）

**为什么AI适合**：
- 营销图像注重视觉冲击力和氛围感
- 文字内容相对较少（通常只有标题+标语）
- 需要快速迭代尝试多种风格
- 创意和美感比精确性更重要

**最佳实践**：
1. **明确目标受众**：说明目标人群（如"年轻女性"、"科技从业者"）
2. **指定平台风格**：不同平台有不同风格（小红书vsLinkedIn）
3. **强调品牌色**：使用品牌主色调提升识别度
4. **包含行动号召**：明确添加CTA按钮或文字
5. **测试多个版本**：生成3-5个版本选择最佳

---

### 创意设计类场景（AI强烈推荐 ⭐⭐⭐⭐⭐）

**为什么AI适合**：
- 创意设计需要灵感和多样性
- AI能快速生成多种风格供参考
- 视觉表现比精确性更重要
- 可以快速迭代优化

**最佳实践**：
1. **提供参考风格**：说明想要的设计风格（如"扁平化"、"3D"、"手绘风"）
2. **指定配色方案**：明确主色和辅助色
3. **保持简洁**：Logo和图标设计越简洁越有效
4. **考虑应用场景**：说明使用场景（如App图标、网站Logo）
5. **批量生成**：一次生成多个版本选择

---

### 教育培训类场景（AI谨慎使用 ⭐⭐⭐）

**部分适合**：
- ✅ 教学插图（场景化、示意图）
- ✅ 知识卡片（名言、概念卡片）
- ⚠️ 思维导图（只适合关键词版本）
- ❌ 详细步骤说明（文字多效果差）

**最佳实践**：
1. **限制文字量**：每张图文字不超过50字
2. **使用图标**：用图标替代文字说明
3. **分步骤展示**：复杂过程分成多张图
4. **简化表达**：只保留核心关键词
5. **验证可读性**：生成后检查文字是否清晰

---

### 数据可视化类场景（AI不推荐 ⭐）

**为什么AI不适合**：
- 数据图表需要精确性
- AI无法生成准确的坐标和比例
- 文字渲染容易出错
- 无法保证数据正确性

**推荐替代方案**：
- Excel/WPS表格 → 制作精确对比表格
- ECharts/D3.js → 生成交互式数据图表
- Tableau/PowerBI → 专业数据可视化
- Draw.io/ProcessOn → 手动绘制架构图

**微信公众号封面（特殊要求）**

**为什么特殊**：
- 一张图用于两个场景（头条 + 分享）
- 分享时会自动裁剪为正方形
- 必须确保关键内容在中心区域

**关键要素**：
1. **比例**：2.35:1 横版（900x383）
2. **安全区域**：关键内容放在中心60%区域
3. **边缘处理**：四周留白，不放置关键元素
4. **双重优化**：确保两个场景都清晰可读

**设计原则**：
```
中心60%：安全区（所有关键内容）
四周各20%：可能被裁剪区（只放装饰元素）
```

**模板**：
```
微信公众号封面图（2.35:1，900x383）

中心区域（安全区）：
- 主标题：{核心标题，居中}
- 副标题：{简短补充}
- 核心图标：{主题图标}

背景：
{渐变色}

边缘装饰：
{非关键装饰，可被裁剪}
```

**最佳实践**：
1. 使用网格辅助设计，划分中心60%安全区
2. 主标题精炼，控制在10字以内
3. 重要图标和文字必须居中
4. 避免文字延伸到画面边缘
5. 生成后测试裁剪效果

## 💰 成本控制策略

### 1. 开发阶段优化

**策略**：
- 使用 `quality="standard"`
- 使用较小尺寸 `1024x1024`
- 先单张测试，满意后再批量

**预期节省**：50-70%

---

### 2. 批量生成优化

**策略**：
- 使用 `batch_generate` 方法
- 设置合理的 `delay`（1-2秒）
- 相似prompt复用结果

**示例**：
```python
# 不好的做法（每次独立请求）
for prompt in prompts:
    result = generator.generate_image(prompt)

# 好的做法（批量生成）
results = generator.batch_generate(prompts, delay=1.5)
```

---

### 3. 智能缓存

**策略**：
- 保存生成的图像URL
- 相似prompt使用已有结果
- 建立本地素材库

**实现**：
```python
import hashlib

def get_cache_key(prompt, size, quality):
    content = f"{prompt}_{size}_{quality}"
    return hashlib.md5(content.encode()).hexdigest()

# 检查缓存
cache_key = get_cache_key(prompt, size, quality)
if cache_key in cache:
    return cache[cache_key]
else:
    result = generator.generate_image(prompt)
    cache[cache_key] = result
    return result
```

## 🔍 质量检查清单

### Prompt质量检查

生成前，确保prompt包含：

- [ ] 明确的主体（是什么图）
- [ ] 具体的内容（展示什么）
- [ ] 清晰的布局（如何组织）
- [ ] 定义的风格（视觉效果）
- [ ] 必要的细节（颜色、比例、背景）

---

### 生成结果检查

生成后，检查图像质量：

- [ ] 文字是否清晰可读
- [ ] 颜色是否符合预期
- [ ] 布局是否合理
- [ ] 内容是否完整
- [ ] 风格是否统一

---

### 优化建议

**如果生成结果不理想**：

1. **文字不清晰/有错别字**：
   - **营销/创意类**：减少文字内容，只保留标题
   - **技术图表类**：**强烈建议改用专业工具**（Excel/PPT/Draw.io）
   - **教育培训类**：改用图标替代文字说明
   - 如果必须使用文字：添加"大字号"、"清晰字体"、"无错别字"
   - 考虑生成后手动添加文字（使用PS/Canva）

2. **颜色不符**：
   - 明确指定颜色名称（蓝色、绿色、橙色）
   - 使用"渐变色"增强视觉效果
   - 添加"鲜艳"、"柔和"等修饰词

3. **布局混乱**：
   - 明确指定"横向排列"、"纵向排列"
   - 使用数字标记顺序
   - 描述具体位置（左侧、右侧、中间）

4. **内容缺失**：
   - 列出所有必需元素
   - 使用"包含"明确说明
   - 分步骤详细描述

```
  │      理由：创意多样，快速试错，降低门槛
  │      工具：AI生图 → Figma/Illustrator精修
  │
  ├─ 教育培训（教学图、卡片）
  │   └─ ⚠️ 谨慎使用AI生图
  │      决策条件：
  │      ├─ 文字极少（<50字）→ ✅ 可用AI
  │      ├─ 主要是图标/插图 → ✅ 可用AI
  │      └─ 需要详细说明 → ❌ 用专业工具
  │
  ├─ 内容创作（封面、配图）
  │   └─ ✅ 推荐AI生图
  │      理由：氛围感强，视觉效果好
  │
  └─ 技术文档（架构图、对比表、代码）
      └─ ❌ 不推荐AI生图
         理由：文字多、精确性要求高
         推荐工具：Excel/PPT/Draw.io/ECharts
```
