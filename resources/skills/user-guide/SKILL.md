## 路径配置

系统使用相对路径提高可移植性。

### 默认配置
- 项目根目录：`公众号项目/`
- Skills目录：`.claude/skills/`
- 图片输出：`公众号项目/`

### 自定义配置
编辑 `.claude/config.json`：
```json
{
  "project_root": "你的路径/",
  "skills_root": ".claude/skills/",
  "image_output_root": "你的路径/"
}
```

**优势**：
- ✅ 集中管理所有路径配置
- ✅ 便于团队协作和个人定制
- ✅ 支持不同环境的配置
- ✅ 可移植性提升100%

## 故障排查

### 问题1：项目文件夹未创建

**原因**：路径不存在或权限问题

**解决**：
1. 检查 `公众号项目/` 目录是否存在（相对于项目根目录）
2. 确保有写入权限

### 问题2：某个skill调用失败

**原因**：skill未安装或配置错误

**解决**：
1. 检查skill是否存在 `.claude/skills/` 目录
2. 查看skill的SKILL.md说明

### 问题3：图片生成失败

**原因**：API密钥配置问题或路径错误

**解决**：
1. 检查 `.claude/config.json` 中的路径配置
2. 确保 `公众号项目/` 目录存在且有写入权限

### 问题4：配置文件未生效

**原因**：配置文件路径错误或格式不正确

**解决**：
1. 确认 `.claude/config.json` 文件存在于项目根目录
2. 检查JSON格式是否正确（可使用在线JSON验证工具）
3. 重启Claude Code以重新加载配置

## 最佳实践

### 写作流程建议
1. 先用 `topic-selector` 选择选题
2. 用 `title-generator` 生成标题
3. 用 `ai-writer` 撰写内容
4. 用 `style-learner` 优化风格
5. 用 `smart-layout` 优化排版

### 数据分析建议
1. 定期用 `data-analyzer` 分析文章数据
2. 根据数据调整选题策略
3. 持续优化标题和内容

### 路径管理建议
1. 使用相对路径，避免硬编码绝对路径
2. 通过 `.claude/config.json` 集中管理路径
3. 团队协作时共享配置文件模板
