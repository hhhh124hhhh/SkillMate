## 示例4: 完整工作流

### Step 1: 写作

用户写完文章"死了么?应用开发者的困局"。

### Step 2: emotion-provoker优化

```
/emotion-provoker
我写了一篇关于"死了么"的文章,感觉太理性了
```

emotion-provoker指出问题,用户修改文章。

### Step 3: emotion-card-generator生成卡片

```
/emotion-card-generator
帮我为改写后的文章生成情绪卡片
```

emotion-card-generator提取情绪点,生成5张卡片。

### Step 4: 发布

用户:
- 发布文章到公众号
- 分享情绪卡片到小红书/Instagram

## 示例6: 错误处理

### 场景1: 文章太短

**用户输入**:
```
/emotion-card-generator
帮我生成情绪卡片

文章内容:
"今天天气不错。"
```

**Skill回应**:
```

这篇文章太短了(只有8个字),无法提取足够的情绪点。

建议:
- 文章至少500字
- 包含真实的情绪表达

你可以:
1. 先用 /emotion-provoker 帮你扩展文章
2. 或者等文章写完再生成卡片
```

---

### 场景2: 文章太理性

**用户输入**:
```
/emotion-card-generator
帮我生成情绪卡片

文章内容:
"应用开发需要考虑技术栈、用户体验、商业模式等多个因素..."
```

**Skill回应**:
```

这篇文章比较理性,缺少个人情绪表达。

我尝试提取,但只找到2个情绪点:
1. "应用开发需要考虑..."(非情绪,未提取)
2. [未找到其他情绪点]

建议:
1. 先用 /emotion-provoker 帮你加入真实情绪
2. 然后再生成卡片

emotion-provoker会帮你写出真实的感受。
```
