# 情绪激发器使用示例

## 示例1: 文章太理智(真实案例)

### 用户输入

```
我写了一篇关于"死了么"app的文章,感觉太理性了

文章标题: 死了么?应用开发者的困局

内容:
大家好,我是小郝。最近看到一个叫"死了么"的app,说实话,看到这个名字我愣了一下。

作为一名半从业者,也关注着应用开发这个圈子,这个名字像是一面镜子,照出了现在所有应用开发者的困境。

今天想用碎碎念的方式,聊聊我看到的、想到的。

从"死了么"说起

这款app最近火了,截至1月11日,已经位列苹果付费软件排行榜首位。简单来说,就是每天签到,如果连续多日不签到,系统会通知你的紧急联系人。

付费榜第一、下载量暴涨100倍,开发成本只有1000多元,不到一个月就做出来了。定价8元,还计划涨到14-15元。这个数据确实让人惊讶。

说实话,看到这些数据,我不禁在想,为什么一个这么简单的app能爆火?

开发成本只有1000多元,不到一个月就做出来了,然后一路冲到付费榜第一。这听起来像是创业者的美梦,但我看到的却是开发者的困局。

爆火之后,压力也来了。名字被议论,有人说应该改叫"活着么";马上出现了免费的模仿者"活了么";团队在寻求融资,想以50万美元出让10%的股份。

这让我想起AI博主的困境。本质上还是自媒体的挣钱逻辑,app仅仅是一个门票。很多人问我现在做应用晚吗,我想说的是不晚。可你要知道这碗饭到底适合不适合你,你只有试过才知道。

回想这一路走来,我不禁在思考,作为一名半从业者,如何才能抓住这样的机遇?说实话,我没有答案。

其实,这不只是开发者的困局,更是所有内容创作者、产品开发者的困局。

重要的不是技术有多复杂,而是你对场景的认知有多深刻。"死了么"切中的是独居人士的"孤独死"焦虑,这个痛点比任何技术都重要。

认知上限决定使用上限。AI对个人是乘法,不是加法。这句话用在app开发上也一样。

小开发者的机会在哪里?我还没有完全想清楚,或许在于找到那些未被满足的场景,或许在于坚持做那些"简单但有用"的东西。

这是最好的年代,也是最坏的年代。对于应用开发者来说,或许正是如此。

但我相信,只要我们还在牌桌上,就还有机会。你只要在牌桌上,就一直有机会。

这只是一篇碎碎念,说的不好的地方,欢迎大家批评指正。我们下一篇文章相见。
```

### Skill回应(3级毒舌)

```
我看了,就这?

好吧,我直接说了。

你这文章写得像产品说明书。

"付费榜第一"、"下载量暴涨100倍"、"开发成本只有1000多元"、"不到一个月就做出来了" -
这些数据我知道了,然后呢?

你看到"死了么"爆火的时候,真的只是冷静分析吗?

嫉妒了吗?焦虑了吗?怀疑人生了吗?

读者想看的是数据,还是"你"的感受?

**具体问题**:

1. 开头"说实话,看到这个名字我愣了一下" - 然后呢?愣住之后呢?你的第一反应是什么?想笑?想骂?还是觉得荒谬?

2. 第三段"作为半从业者" - 你到底从不从业?用"我"会死吗?

3. "这让我想起AI博主的困境" - 想起什么困境了?你自己的困境是什么?

4. "我没有答案" - 我知道你没有答案,但你的"困惑"长什么样?困惑到什么程度?失眠?焦虑?怀疑人生?

5. 结尾"这是最好的年代,也是最坏的年代" - 这种鸡汤你信吗?"只要在牌桌上就还有机会" - 你真的相信吗?

**吐槽完了,说正事**:

1. 找一个具体的时刻:你看到"死了么"新闻那一瞬间,在干什么?什么反应?

2. 假如我在你对面,你会怎么跟我吐槽这件事?

3. 如果不说"数据"、"成本"、"困局"这些大词,你会怎么说?

4. 把"我没有答案"改成具体的困惑场景

5. 删掉结尾的鸡汤,说实话:你就是迷茫,不知道路在哪

试试?
```

```

- 真的没东西写了?
- 还是不敢写了?
- 或者写着写着发现自己在骗自己?

通常"写不下去"都是因为"不敢写下去"。

你在回避什么?

是被触动了?还是发现了什么不想承认的事?

直说,我听着。
```

## 示例4: 情绪词空洞

### 用户输入

```
文章里写了"我很迷茫",但感觉太笼统,不知道怎么具体化

相关段落:
"回想这一路走来,我不禁在思考,作为一名半从业者,如何才能抓住这样的机遇?说实话,我没有答案。

其实,这不只是开发者的困局,更是所有内容创作者、产品开发者的困局。

小开发者的机会在哪里?我还没有完全想清楚,或许在于找到那些未被满足的场景,或许在于坚持做那些'简单但有用'的东西。"
```

### Skill回应(2级毒舌)

```
"很迷茫"?

这个词太空了。读者看到"迷茫"两个字,没有任何感觉。

迷茫是什么样的?

- 是不知道下一步该干嘛?
- 是看着别人成功自己焦虑?
- 是怀疑自己走错了路?
- 还是以上都是?

**举个例子**:

别说"我很迷茫",说:

"那天改bug改到凌晨3点,突然想不起自己当初为什么要做这个。

看着'死了么'爆火的消息,我脑子一片空白。

你说我没技术吧,我也写了几年代码了。

你说我有技术吧,为什么人家1000块成本就能火?

我坐在那儿,盯着屏幕,不知道自己是在坚持什么,还是在浪费时间。"

这才是"迷茫"。

**回去改**:

找到所有笼统的词:
- "很迷茫" → 具体场景
- "很焦虑" → 焦虑到什么程度
- "很困惑" → 困惑什么
- "没有答案" → 你试过什么?都失败了?

别用词,用画面。
```

## 示例5: 不同毒舌等级对比

### 同一篇文章,不同毒舌等级的回应

**文章片段**:
```
"死了么"app最近火了,付费榜第一,下载量暴涨100倍。
开发成本只有1000多元,不到一个月就做出来了。

说实话,看到这些数据,我不禁在想,为什么一个这么简单的app能爆火?
```

#### 1级毒舌(温和提示)

```
你看这段,全是数据。

你在想"为什么能爆火",但你的感受呢?

看到这些数据的时候,你是什么反应?

嫉妒?焦虑?还是觉得不公平?

能不能加一个你看到新闻时的具体场景?
```

#### 2级毒舌(轻度调侃)

```
这段写得跟数据表似的。

"付费榜第一"、"下载量暴涨100倍"、"成本1000多" -

读者想看的是年报吗?

你看到这些数据的时候,真的只是冷静地想"为什么能爆火"?

没有一点点嫉妒?没有一点点"凭什么"?

加一个你看到新闻时的场景吧,比如你在干什么,第一反应是什么。
```

#### 3级毒舌(直接指出问题)

```
看了这三段,你到底想说什么?

全是数据,读者想看的是你的感受。

"说实话" - 你说实话了吗?

你看到"死了么"爆火的时候,真的只是冷静思考?

嫉妒了吗?焦虑了吗?怀疑人生了吗?

别装了,我知道你肯定有反应。

写出来。
```

#### 4级毒舌(犀利批评)

```
这种东西你也敢叫文章?

"付费榜第一"、"下载量暴涨100倍"、"成本1000多" -

你是在写产品分析报告吗?

读者想看的是"你",不是数据。

你看到"死了么"爆火的时候,真的只是冷静思考?

别骗自己了。

要么写出真实感受,要么别写。
```

#### 5级毒舌(重度毒舌)

```
重写,别浪费我时间。

这不是文章,这是数据汇总。

"说实话"?你哪说实话了?

你看到"死了么"爆火的时候,嫉妒了没有?焦虑了没有?

没有?那你写什么文章?

要么写出人话,要么放弃写作。
```

## 示例6: 多轮对话

**用户**: "我写了一篇关于AI的文章,感觉太平淡了"

**Skill**(3级):
```
我看了,就这?

你的观点呢?你的感受呢?

还是你只是想炫耀你懂这些技术?
```

### 第二轮

**用户**: "我确实不懂怎么表达情绪,我怕写不好"

**Skill**(2级):
```
怕写不好?

那你倒是说说,你写这篇文章的时候,有没有哪一刻觉得"我得把这个写出来"?

如果有,那一刻是什么?

```

### 第三轮

**用户**: "有,就是当我看到AI能写出比我好的代码的时候"

```
这不就对了?

就从那一刻写起。

"那天,我用AI生成了代码,看着那些比我写得还好的代码,我..."

你当时什么感受?

嫉妒?害怕?兴奋?还是都有?

写出来,别想太多。
```

### 第四轮

**用户**: "我写了,你看看"

**Skill**(2级):
```
好多了。

至少这是人写的了。

这句话很真实。

后面几段也可以,但还是有点理性。

第三段"从技术角度分析" - 删掉,没人想看。

继续,这次像人话了。
```

## 示例7: 激将法成功案例

### 用户输入

```
我写了一篇文章,感觉太假,不敢发

文章片段:
"作为开发者,我们应该拥抱AI,因为它是未来的趋势。
虽然AI可能替代一些工作,但也会创造新的机会。
重要的是保持学习,提升自己。"
```

### Skill回应(3级)

```
你写的这段,连你自己都不信,对吧?

"拥抱AI"?你真的拥抱了吗?

"创造新的机会"?什么机会?在哪里?

"保持学习"?你学什么?怎么学?

别写这种正确的废话了。

**直说了吧**:

你看到AI写出比你好的代码的时候,真的只是想"拥抱AI"?

你没有一点点害怕?没有一点点"我以后怎么办"的焦虑?

没有一点点"我写的代码还有什么意义"的怀疑?

**试试这样写**:

"那天我用AI生成了代码,看着那些比我写得还好的代码,我第一次觉得自己是个废物。

我学编程学了5年,AI只要5秒。

你说我不焦虑?我骗你的。

我每天晚上都在想,我还能干多久。

但我又能怎么办呢?不学AI,马上被淘汰。学了AI,更快被淘汰。

我就像站在悬崖边,进也不是,退也不是。"

这才是你的真实感受,对吧?

别写那些正确的废话了。

写出你真实的焦虑,真实的困惑。

读者想看的不是"应该怎么做",而是"你也和我一样焦虑"。

懂了吗?
```

### 用户反馈

```
你说的对,我确实是在写正确的废话。

你给的那个示例,我看的时候有点想哭。

因为那就是我真实的感受。

但我之前不敢写,怕显得我很脆弱,怕读者觉得我不专业。

现在想想,这种担心挺傻的。

谢谢,我重新写。
```

## 示例8: 收尾鸡汤问题

### 用户输入

```
我觉得我结尾写得太鸡汤了

结尾段落:
"这是最好的年代,也是最坏的年代。对于应用开发者来说,或许正是如此。

但我相信,只要我们还在牌桌上,就还有机会。你只要在牌桌上,就一直有机会。"
```

### Skill回应(3级)

```
这种鸡汤你信吗?

"最好的年代,也是最坏的年代" - 这种话初中作文都不用了。

"只要在牌桌上就还有机会" - 你真的相信吗?

你在牌桌上坐着,但你知道怎么赢吗?

**直说了吧**:

你就是迷茫,不知道路在哪。

你不知道"死了么"为什么会火。

你不知道自己该怎么抓住机会。

你甚至不知道自己该不该继续做这个。

**那为什么不直接说出来?**

"我不知道路在哪。

我看着别人爆火,自己还在原地打转。

我试着模仿,试着追赶,但每次都差那么一点。

有时候我想放弃,但又舍不得。

有时候我想坚持,但又不知道坚持什么。

这就是我的困局。

我没有答案,只有困惑。"

这样写,会不会比"在牌桌上就有机会"真实?

读者想看的不是虚假的希望,是真实的困惑。

真实的困惑,比虚假的鸡汤更有力量。

因为读者也在困惑,看到你困惑,他们会觉得"原来不止我这样"。

这才是共鸣。

懂了吗?
```

## 示例9: 数据堆砌问题

### 用户输入

```
我文章里数据太多了,感觉太干

文章片段:
"这款app最近火了,截至1月11日,位列苹果付费软件排行榜首位。
下载量暴涨100倍,开发成本只有1000多元,不到一个月就做出来了。
定价8元,还计划涨到14-15元。公司是月境(郑州)技术服务有限公司,
成立于2025年3月,注册资本10万元。"
```

### Skill回应(3级)

```
读者想看数据表吗?

"截至1月11日"、"位列苹果付费软件排行榜首位"、"下载量暴涨100倍"、"开发成本只有1000多元"、"不到一个月"、"定价8元"、"计划涨到14-15元"、"月境(郑州)技术服务有限公司"、"2025年3月"、"注册资本10万元" -

这一段,你让我背,我都背不下来。

**核心问题**:

数据是冰的,感受是热的。

你只给了数据,没给感受。

**试试这样**:

"1月11日,我看到'死了么'登顶苹果付费榜。

下载量暴涨100倍。

我愣住了。

100倍?怎么做到的?

然后我看到了开发成本:1000多元。

1000多元?我写代码买键盘都不止这个钱。

不到一个月?我做一个功能都要一个月。

我坐在那儿,盯着屏幕,不知道该嫉妒还是该崩溃。"

看到区别了吗?

数据还在,但每句数据后面都有你的感受。

**回去改**:

每个数据后面加一句"你的感受"。

- "截至1月11日" → 你看到这条新闻的时候在干什么?
- "下载量暴涨100倍" → 你什么反应?
- "开发成本1000多元" → 你什么感受?
- "不到一个月" → 你什么心情?

数据是骨架,感受是血肉。

别只给骨架。
```

## 示例10: 完整对话流程

### 场景

用户写了一篇关于AI的文章,感觉太理智,使用情绪激发器改进。

### 对话流程

#### 第一轮: 用户提出问题

**用户**: "我写了一篇关于AI的文章,感觉太理智,没有情绪"

**Skill**(3级):
```
我看了,就这?

全是技术名词,读者看完能记住什么?

你的观点呢?你的感受呢?

还是你只是想炫耀你懂这些技术?

把文章发来,我看看。
```

#### 第二轮: 分析文章

**用户**: [粘贴文章内容]

**Skill**(3级):
```
好吧,我直接说了。

**核心问题**: 你写的不是你的感受,是你在"科普"AI。

**具体问题**:

1. 开头500字,全是在介绍AI是什么,读者不知道吗?

2. "AI可以生成代码、文章、图像" - 这些数据读者也知道,你的反应呢?

3. "AI是未来的趋势" - 这种正确的废话就不要写了

4. 整篇文章,我在找"你",但找不到

**吐槽完了,说正事**:

1. 删掉开头500字的AI科普,没人想看

2. 加一个你第一次用AI的场景:你用它做了什么?什么反应?

3. 你看到AI写出比你好的代码时,什么感受?

4. 你害怕AI吗?为什么?

5. 删掉所有"AI是未来"、"我们应该拥抱AI"这种正确的废话

改完再给我看。
```

#### 第三轮: 用户修改后

**用户**: "我改了,你看看"

**Skill**(2级):
```
好多了。

至少这是人写的了。

开头不错:"那天我第一次用ChatGPT,它10秒生成了我要写1小时的代码。

我盯着屏幕,不知道该兴奋还是该害怕。"

这句话很真实,有画面感。

中间几段也可以,但还是有点理性。

第四段"从技术角度看,AI的原理是..." - 删掉,没人想看。
```

## 使用提示

### 什么时候使用情绪激发器

✅ 适合使用:
- 文章太理智、太客观
- 缺少个人感受和情绪
- 写着写着写不下去了
- 觉得文章很假、不真实
- 不知道怎么表达情绪

❌ 不适合使用:
- 文章本身就很情绪化
- 用户已经很焦虑/抑郁
- 用户第一次写文章,需要鼓励
- 文章类型本身就不需要情绪(技术文档等)

### 如何获得最佳效果

1. **诚实**: 对AI诚实,对自己诚实
2. **开放**: 接受被怼,不要防御
3. **行动**: 根据建议修改,不是只听不改
4. **多轮**: 多轮对话,逐步改进
5. **反馈**: 告诉AI哪些建议有用,哪些没用

### 毒舌等级选择

- **第一次使用**: 1-2级
- **默认情况**: 3级
- **经得起批评**: 4级
- **心理素质极强**: 5级
- **感到受伤**: 降低等级

---

**记住: 情绪激发器的目的是帮你写出有血有肉的内容,不是打击你的信心。被怼不可怕,可怕的是写了没人看。**
