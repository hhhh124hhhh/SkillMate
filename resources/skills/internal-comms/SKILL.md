# 内部通信工具

## 概述

按照公司喜欢的格式编写内部通信内容。

**支持类型**：
- 3P 更新（进展、计划、问题）
- 公司简报
- FAQ 回答
- 状态报告
- 领导更新
- 项目更新
- 事故报告

## 何时使用

**使用此技能**编写：
- 团队状态更新
- 周报/月报
- 项目进展报告
- 问题解答
- 领导层汇报
- 公司内部新闻
- 事故报告

## 3P 更新（进展、计划、问题）

### 格式

```markdown
# 团队名称 - 周报

**日期**：YYYY-MM-DD
**报告人**：姓名

## 进展 (Progress)
- ✅ 完成的项目/任务 1
- ✅ 完成的项目/任务 2
- 🚧 进行中的工作 3（完成度 70%）

## 计划 (Plans)
- 📋 下周计划 1
- 📋 下周计划 2
- 🎯 季度目标进展

## 问题 (Problems)
- ⚠️ 阻塞问题 1（需要支持）
- ⚠️ 风险/挑战 2
- 💡 建议/请求

## 下一步
- [ ] 行动项 1
- [ ] 行动项 2
```

### 示例

```markdown
# 工程平台团队 - 周报

**日期**：2025-01-20
**报告人**：张三

## 进展 (Progress)
- ✅ 完成用户认证系统重构（减少延迟 40%）
- ✅ 部署 CI/CD 管道改进（构建时间减少 25%）
- 🚧 微服务迁移进行中（完成度 60%，3/5 服务已迁移）

## 计划 (Plans)
- 📋 完成剩余 2 个服务的迁移
- 📋 开始性能优化阶段
- 🎯 Q1 目标：所有核心服务迁移完成

## 问题 (Problems)
- ⚠️ 数据库连接池偶尔耗尽（需要 DBA 支持）
- ⚠️ 旧 API 文档不完整，影响迁移进度
- 💡 建议：增加自动化测试覆盖

## 下一步
- [ ] 与 DBA 团队会议（周二 10:00）
- [ ] 完成服务 4 迁移测试
- [ ] 更新 API 文档
```

## 最佳实践

### 1. 清晰简洁

**好的**：
```
✅ 完成用户认证系统（延迟降低 40%）
```

**不好的**：
```
❌ 我们一直在做用户认证系统的工作，
   现在基本完成了，性能应该好了一些
```

### 2. 使用状态指示器

- ✅ 完成
- 🚧 进行中
- 📋 计划中
- ⚠️ 问题/风险
- 🎯 目标
- 🟢 正常
- 🟡 有风险
- 🔴 严重问题

### 3. 可操作的更新

**好的**：
```
⚠️ 数据库连接池耗尽
需要：DBA 团队审查配置
行动：安排周二会议
```

**不好的**：
```
❌ 数据库有点问题，我们正在看
```

### 4. 量化和具体

**好的**：
```
✅ 性能提升：延迟降低 40%（500ms → 300ms）
✅ 覆盖率：测试覆盖从 60% 提升到 85%
```

**不好的**：
```
❌ 性能好多了
❌ 测试增加了
```

## 写作技巧

### 标题和摘要

- 使用清晰、描述性的标题
- 执行摘要限制在 2-3 句话
- 将最重要的信息放在前面

### 结构

- 使用标题分隔部分
- 使用项目符号列出清单
- 使用表格展示数据
- 使用加粗强调关键点

### 语气

- 专业但不过于正式
- 直接明确，避免模糊
- 客观陈述事实
- 建设性提出问题

### 时间和日期

- 使用 ISO 格式（YYYY-MM-DD）
- 明确时间范围（本周、下周）
- 标注截止日期

## 常见场景

### 场景 1：周报更新

```markdown
# 工程团队 - 第 3 周更新

本周完成了 API 重构，开始下一阶段的性能优化。
遇到一些数据库问题，需要 DBA 支持。

详见：
- 进展：3 个主要任务完成
- 问题：1 个阻塞问题
- 下周：2 个计划任务
```

### 场景 2：项目里程碑

```markdown
# 里程碑更新：Phase 1 完成

🟢 状态：完成

我们很高兴地宣布，项目 Phase 1 已完成！
核心功能全部就绪，测试覆盖率达标。

下一步：Phase 2 准备工作
```

### 场景 3：问题升级

```markdown
# ⚠️ 问题升级：生产环境性能

🔴 严重性：P1

影响：
- 用户响应时间增加 300%
- 错误率上升至 5%

已采取的行动：
- 回滚最近部署
- 启用缓存

需要：
- 架构团队审查
- 性能分析支持
```

## 关键原则

- **清晰明确**：使用具体语言和数字
- **结构化**：标题、项目符号、表格
- **可操作**：明确的下一步和责任人
- **诚实透明**：如实报告问题和风险
- **简洁专业**：尊重读者时间
- **频繁更新**：定期沟通进展
