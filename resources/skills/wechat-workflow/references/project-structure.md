# 公众号项目文件夹结构规范

本文档详细说明公众号文章项目的文件夹结构、文件命名规范和内容要求。

---

## 📁 标准项目结构

```
公众号项目/
└── [文章标题]/
    ├── 00_背景资料.md       # 热点事件、技术数据、参考资料
    ├── 01_文章大纲.md       # 详细写作大纲
    ├── 02_文章内容.md       # 完整文章内容
    ├── 03_发布记录.md       # 发布数据、用户反馈
    └── assets/              # 图片、图表、素材
        ├── images/          # 文章配图
        ├── charts/          # 数据图表
        └── references/      # 参考资料
```

---

## 📄 文件说明

### 00_背景资料.md

**用途**：收集和整理所有背景资料

**内容结构**：
```markdown
# 背景资料 - [文章标题]

> 更新时间：YYYY-MM-DD

## 核心要素
- 文章标题
- 核心话题
- 目标受众
- 预计字数

## 热点事件
- 事件名称、时间、地点
- 关键数据
- 时间线
- 官方声明

## 技术对比
- 技术指标
- 基准测试
- 价格对比
- 适用场景

## 用户独特价值
- 真实项目经历
- 获奖或成就
- 具体数据

## 参考资料
- 文章链接
- 数据来源
```

**示例**：见当前项目的 `00_背景资料.md`

---

### 01_文章大纲.md

**用途**：详细的文章写作大纲

**内容结构**：
```markdown
# 文章大纲 - [文章标题]

## 一、开头（约400字）
- 1.1 热点引入
- 1.2 个人经历
- 1.3 价值预告

## 二、项目介绍（约600字）
- 2.1 项目背景
- 2.2 核心创新
- 2.3 项目成果

...（共7部分）

## 字数分配
| 部分 | 字数 | 占比 |
...
```

---

### 02_文章内容.md

**用途**：完整的文章内容

**内容结构**：
```markdown
# [文章标题]

---

## 开头

[正文内容...]

---

## 项目介绍

[正文内容...]

...

---

## 互动引导

如果你有类似的使用体验，欢迎交流...

---

**写作时间**：YYYY-MM-DD
**预计发布**：YYYY-MM-DD
```

**注意事项**：
- 使用小标题分隔不同部分
- 每段不超过5-6行
- 适当使用加粗、引用等格式
- 插入图片占位符：`![图片说明](assets/images/图片名.png)`

## 📝 文件命名规范

### 项目文件夹命名

**格式**：`[文章标题]`

**规则**：
- 使用文章完整标题或核心关键词
- 避免特殊字符（/ \ : * ? " < > |）
- 长度建议不超过50个字符

**示例**：
- ✅ `智谱上市_GLM文章`
- ✅ `GLM_vs_DeepSeek_深度测评`
- ❌ `GLM/DeepSeek对比`（包含特殊字符）

### 资源文件命名

**图片**：
- `封面图.png`
- `对比表格.png`
- `GLM架构图.jpg`

**图表**：
- `错误率对比.svg`
- `Token消耗趋势.png`

**参考资料**：
- `GLM-4.7技术白皮书.pdf`
- `参考资料链接.txt`

## 🔄 项目生命周期

## 💡 最佳实践

### 1. 版本控制

重要修改时，在文件末尾添加版本记录：

```markdown
---

## 版本历史
- v1.0 (YYYY-MM-DD)：初稿完成
- v1.1 (YYYY-MM-DD)：补充技术数据
- v1.2 (YYYY-MM-DD)：优化标题和结尾
```

### 2. 协作编辑

如果多人协作：
- 在文件开头标注编辑者
- 使用注释标记待讨论内容：`<!-- TODO: 讨论这个观点 -->`
- 重大修改在评论区说明

### 3. 备份策略

- 定期备份整个项目文件夹
- 发布前锁定02_文章内容.md
- 重要版本创建快照：`[文章标题]_v1.0_备份/`

- ✓ 00_背景资料.md 资料充分、来源明确

- ✓ 03_发布记录.md 初始化完成
- ✓ assets/ 目录素材齐全
- ✓ 所有文件使用UTF-8编码
- ✓ 图片尺寸符合要求
- ✓ 无特殊字符导致的文件名问题
