# 公众号工作流管理

整合所有公众号相关工具的协调层skill，提供从项目管理到内容发布的全流程支持。

## 核心功能

1. **项目管理** - 创建标准项目文件夹结构（00_背景资料、01_大纲、02_内容、03_发布记录）
2. **创作协调** - 整合选题、标题、写作、风格、排版等skills完成文章创作
3. **数据分析** - 基于历史数据优化选题和内容策略
4. **竞品分析** - 学习竞品文章策略和写作风格

## 工作流程

### 流程A：完整创作（从零开始）

```
用户：帮我写一篇关于[主题]的文章

执行序列：
1. topic-selector → 推荐热点选题
2. title-generator → 生成候选标题（10+）
3. wechat-writing → 设计文章结构（7部分框架）
4. style-learner → 匹配用户写作风格
5. ai-writer → 创作文章内容
6. smart-layout → 优化排版（6种模板）
7. data-writer → 录入项目数据

输出：公众号项目/[标题]/ 完整文件夹
```

### 流程B：数据驱动优化

```
用户：分析我的文章数据，给出优化建议

执行序列：
1. data-analyzer → 分析历史文章（阅读量、趋势）
2. 识别高阅读量文章特征
3. topic-selector → 优化选题策略
4. title-generator → 优化标题
5. 输出内容优化建议报告
```

### 流程C：竞品分析

```
用户：分析[公众号名称]的文章策略

执行序列：
1. wechat-article-fetcher → 采集竞品文章
2. data-analyzer → 分析内容策略
3. style-learner → 学习写作风格
4. title-generator → 生成竞品标题
5. 输出竞品分析报告
```

## 输出规范

### 项目文件夹结构

每个公众号文章项目遵循以下结构：

```
公众号项目/
└── [文章标题]/
    ├── 00_背景资料.md       # 热点事件、技术数据、参考资料
    ├── 01_文章大纲.md       # 详细写作大纲（7部分框架）
    ├── 02_文章内容.md       # 完整文章内容
    ├── 03_发布记录.md       # 发布数据、用户反馈
    └── assets/              # 图片、图表、素材
```

### 输出文件格式

- **00_背景资料.md**：热点事件、技术对比、用户独特价值、参考资料
- **01_文章大纲.md**：根据文章类型选择合适结构（工具测评/教程指南/观点评论/经验分享）
- **02_文章内容.md**：完整文章，参考2026年写作规范（但灵活应用）
- **03_发布记录.md**：发布日期、阅读量、转发量、用户反馈

### 推荐标准

基于数据分析的推荐标准（但不是硬性要求）：

**强烈推荐**（显著提升效果）：
- ✅ 开头3行包含关键词（点击率 +15-20%）
- ✅ 数据准确有来源（可信度 +40%）
- ✅ 有具体建议（用户价值）

**根据内容类型**（灵活应用）：
- ✅ 配图：图文并茂提升完成率37%，但某些纯文字类型可不需要
- ✅ 金句开篇：适合观点类，教程类可用问题引入
- ✅ 自然分隔：2026年主流趋势，但列表类可用数字编号

**核心原则**：质量 > 形式。好内容比死板遵循规范更重要。

详见：references/writing-standards-2026.md

## 设计权衡说明

### 为什么提供参考框架而非强制模板？

**答案**：类型多样性 > 结构统一性

不同类型文章需要不同结构：
- **工具测评**：开头→介绍→对比→体验→建议→结尾
- **教程指南**：问题→方案→步骤→案例→总结
- **观点评论**：观点→论证→案例→反驳→结论
- **经验分享**：背景→挑战→解决方案→收获→建议

虽然提供参考框架，但强调：
1. **灵活性**：根据主题选择合适结构
2. **多样性**：鼓励创新和实验
3. **质量优先**：好内容比死板结构更重要

参考框架只是起步点，不是终点。

### 为什么要求每部分至少1张配图？

**答案**：2026年内容趋势

根据2026年1月数据分析：
- 图文并茂的文章阅读完成率提升37%
- 配图成为内容的重要组成部分，不是装饰
- AI配图技术成熟，可快速生成

权衡：
- **成本**：AI生成配图，时间成本<5分钟
- **收益**：阅读完成率+37%，分享率+15%
- **决策**：收益远大于成本，要求强制配图

### 为什么要求开头3行包含关键词？

**答案**：移动端阅读习惯

2026年数据显示：
- 60%用户在手机上阅读
- 前3行决定是否继续阅读（3秒法则）
- 关键词前置提升点击率15-20%

权衡：
- **自然度**：可能影响写作流畅性
- **点击率**：提升15-20%
- **决策**：硬性要求，但可通过热点切入自然实现

## 故障排查

### 问题：项目文件夹未创建

**原因**：路径不存在或权限问题

**解决**：
1. 检查 `公众号项目/` 目录是否存在（相对于项目根目录）
2. 确保有写入权限

### 问题：某个skill调用失败

**原因**：skill未安装或配置错误

**解决**：
1. 检查skill是否存在 `.claude/skills/` 目录
2. 查看skill的SKILL.md说明

### 问题：风格匹配不准确

**原因**：历史文章数据不足

**解决**：
1. 确保至少有5篇历史文章
2. 使用data-writer录入更多数据
