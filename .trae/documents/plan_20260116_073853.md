# 测试新的MCP配置

## 目标
测试用户提供的新MCP配置，使用 `uvx mcp-server-fetch` 命令启动网页抓取服务。

## 实现步骤

### 1. 检查依赖
- 检查 `uvx` 是否安装
- 检查 `mcp-server-fetch` 是否可用

### 2. 修改MCP配置文件
- 更新 `mcp.json` 文件，替换现有的fetcher配置
- 使用用户提供的新配置：
  ```json
  {
    "mcpServers": {
      "fetch": {
        "name": "fetch",
        "command": "uvx",
        "args": ["mcp-server-fetch"]
      }
    }
  }
  ```

### 3. 重启应用
- 重启应用以加载新的MCP配置
- 检查应用日志，确认MCP服务是否成功连接

### 4. 测试功能
- 尝试分析微信公众号文章链接
- 验证MCP服务是否正常工作
- 检查 `fetch` 相关工具是否出现在工具列表中

## 技术说明
- 配置文件位置：开发模式下使用项目根目录的 `mcp.json`
- 配置结构符合项目现有的 `MCPServerConfig` 接口定义
- 应用启动时会自动加载此配置并连接到MCP服务器
- 工具会以 `fetch__{tool_name}` 的命名空间出现在工具列表中
