## 修复模拟 IPC 调用问题

### 问题分析
- 错误信息：`Unchecked runtime.lastError: The message port closed before a response was received.`
- 错误信息：`Uncaught (in promise) Error: A listener indicated an asynchronous response by returning true, but the message channel closed before a response was received`
- 根本原因：在浏览器开发模式下，模拟 IPC 调用没有为 `config:set-all` 等调用提供适当的响应

### 修复方案
1. **扩展模拟 IPC 调用处理**
   - 在 main.tsx 文件的 mock IPC 实现中，为 `config:set-all` 添加模拟响应
   - 为其他可能的 IPC 调用添加模拟响应，以防止类似错误

2. **改进错误处理**
   - 确保所有模拟 IPC 调用都返回适当的 Promise 响应
   - 防止消息通道在响应前关闭

### 修复步骤
1. 修改 `src/main.tsx` 文件，扩展模拟 IPC 调用的处理
2. 为 `config:set-all`、`agent:abort` 等调用添加模拟响应
3. 保存文件并验证错误是否已解决

### 预期结果
- 应用在浏览器开发模式下能够正常运行
- 不再出现 IPC 相关的错误
- UI 能够正常加载和交互
