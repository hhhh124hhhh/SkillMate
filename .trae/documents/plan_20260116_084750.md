## 移除本地选题搜索工具，只使用 MCP 搜索能力

### 问题分析
1. **系统仍然使用本地技能**：尽管我们已经移除了 WebSearchProvider，但系统仍然在使用本地的 "选题搜索工具" 技能
2. **MCP 工具执行失败**：尝试执行 `aisearch-mcp-server__chatCompletions` 时失败，显示 "Skill info found? false"
3. **工具调用顺序问题**：系统优先使用本地技能，而不是 MCP 工具

### 解决方案
1. **移除本地选题搜索工具**：
   - 删除 `resources/skills/topic-selector/` 目录，完全移除本地选题搜索能力

2. **确保 MCP 搜索工具正确配置**：
   - 检查 `mcp.json` 中的 MCP 服务器配置
   - 确保 MCP 服务器提供了正确的搜索工具

3. **验证修改**：
   - 运行构建命令确保没有编译错误
   - 启动应用测试 MCP 搜索能力

### 预期结果
修改后，系统将不再使用本地的选题搜索工具，而是直接使用 MCP 服务器提供的搜索能力，避免工具调用冲突和卡住的问题。
