## 移除本地搜索能力，只使用 MCP 搜索能力

### 问题分析
当前系统同时提供两种搜索能力：
1. **本地搜索能力**：通过 `WebSearchProvider` 实现，使用 Bing 搜索引擎，工具名为 `internet__web_search`
2. **MCP 搜索能力**：通过连接到外部 MCP 服务器（如 `aisearch-mcp-server`）获得，工具名格式为 `服务器名__工具名`

从终端输出可以看到，系统执行了 `选题搜索工具`（本地技能），而不是使用 MCP 的搜索能力。

### 修复步骤
1. **修改 MCPClientService.ts**：
   - 移除 `WebSearchProvider` 的导入
   - 移除 `webSearchProvider` 成员变量
   - 在 `constructor` 中移除 `WebSearchProvider` 的初始化
   - 在 `getTools` 方法中移除添加内置搜索工具的代码
   - 在 `callTool` 方法中移除对内置搜索工具的检查

2. **验证修改**：
   - 运行构建命令确保没有编译错误
   - 启动应用查看 MCP 连接状态
   - 测试搜索功能是否只使用 MCP 能力

### 预期结果
修改后，系统将只提供 MCP 服务器的搜索能力，不再包含本地搜索能力，确保所有搜索请求都通过 MCP 服务器处理。
