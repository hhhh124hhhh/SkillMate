# 修复智谱模型API URL配置

## 问题分析

通过分析用户提供的设置示例和之前的错误信息，发现以下关键问题：

1. **API URL不正确**：我们之前使用的是 `https://open.bigmodel.cn/api/coding/paas`，但根据用户提供的设置示例，正确的智谱Claude兼容API URL应该是 `https://open.bigmodel.cn/api/anthropic`

2. **路径拼接问题**：`Anthropic SDK` 会自动在Base URL后面拼接 `/v1/messages`，导致路径错误

3. **配置不匹配**：用户提供的设置示例显示智谱的Claude兼容API URL与我们之前使用的不同

## 解决方案

### 修改 ConfigStore.ts

**修改 `ConfigStore.ts`**：
- 修改 `getApiUrl()` 方法，对于智谱模型（包含GLM、zhipu等关键词），固定使用智谱的Claude兼容API URL `https://open.bigmodel.cn/api/anthropic`
- 对于其他模型，使用用户配置的API URL

### 实现步骤

1. **修改 getApiUrl() 方法**：在 `ConfigStore.ts` 中修改 `getApiUrl()` 方法，根据模型类型返回对应的API URL
2. **测试验证**：重新启动开发服务器，测试智谱模型是否可以正常使用

这样修改后，当使用智谱模型时，系统会自动使用正确的API URL `https://open.bigmodel.cn/api/anthropic`，避免路径拼接错误。
